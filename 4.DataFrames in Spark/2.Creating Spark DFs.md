# Step 1: Create SparkSession

Starting from Spark 2.0, **SparkSession** is the entry point for all Spark applications.

---

## Import SparkSession

```python
from pyspark.sql import SparkSession
```

---

## Create SparkSession

```python
spark = SparkSession.builder \
        .appName("DataFrameExample") \
        .getOrCreate()
```

---

## Why SparkSession?

Before Spark 2.0:

```text
SparkContext
SQLContext
HiveContext
```

had to be created separately.

---

After Spark 2.0:

```text
SparkSession
      ↓
Contains
      ↓
SparkContext
SQLContext
HiveContext
```

Everything is available through a single object.

# Method 1: Create DataFrame from Existing RDD

---

## Step 1: Get SparkContext

Since SparkSession already contains SparkContext:

```python
sc = spark.sparkContext
```

---

## Step 2: Read File as RDD

```python
studentsRDD = sc.textFile("students_data.txt")
```

---

## Check Type

```python
type(studentsRDD)
```

Output:

```python
pyspark.rdd.RDD
```

---

## View Data

```python
studentsRDD.collect()
```

Output:

```text
101 A Ravi Hyderabad
102 B Kiran Chennai
103 A Sneha Bangalore
```

Notice:

Each row is treated as a single string.

---

## Step 3: Tokenize Records

```python
studentsRDD = studentsRDD.map(
    lambda x: x.split(" ")
)
```

---

Output

```python
[
 ['101','A','Ravi','Hyderabad'],
 ['102','B','Kiran','Chennai']
]
```

---

## Step 4: Convert RDD to DataFrame

Using:

```python
toDF()
```

---

```python
studentsDF = studentsRDD.toDF(
    ["RollNo","Section","Name","City"]
)
```

---

## Display DataFrame

```python
studentsDF.show()
```

Output:

```text
+------+-------+------+---------+
|RollNo|Section|Name  |City     |
+------+-------+------+---------+
|101   |A      |Ravi  |Hyderabad|
|102   |B      |Kiran |Chennai  |
+------+-------+------+---------+
```

---

# Visualization

# Method 2: Create DataFrame from Collection

---

Suppose we have:

```python
sampleData = [
(101,"Ravi","A","Hyderabad"),
(102,"Kiran","B","Chennai"),
(103,"Sneha","A","Bangalore")
]
```

---

## Create DataFrame

```python
df = spark.createDataFrame(
    sampleData,
    schema=["RollNo","Name","Section","City"]
)
```

---

## Display Data

```python
df.show()
```

Output:

```text
+------+-----+-------+---------+
|RollNo|Name |Section|City     |
+------+-----+-------+---------+
|101   |Ravi |A      |Hyderabad|
|102   |Kiran|B      |Chennai  |
+------+-----+-------+---------+
```

---

# Visualization

```text
Python List

[
 (101,"Ravi","A"),
 (102,"Kiran","B")
]

       ↓

createDataFrame()

       ↓

Spark DataFrame
```

# Method 3: Create DataFrame from CSV File

This is the most commonly used method in real-world Data Engineering.

---

# Step 1: Import Data Types

```python
from pyspark.sql.types import *
```

---

# Step 2: Define Schema

Suppose CSV contains:

```text
101,A,Ravi,Hyderabad
102,B,Kiran,Chennai
```

---

Define columns manually:

```python
schema = StructType([
    StructField("RollNo", IntegerType(), True),
    StructField("Section", StringType(), True),
    StructField("Name", StringType(), True),
    StructField("City", StringType(), True)
])
```

---

# Understanding Schema Components

---

## StructType

Represents the complete DataFrame structure.

```text
DataFrame Schema
```

---

## StructField

Represents a single column.

Example:

```python
StructField(
"RollNo",
IntegerType(),
True
)
```

---

Meaning:

```text
Column Name : RollNo
Data Type   : Integer
Nullable    : True
```

---

# Step 3: Read CSV

```python
df = spark.read.csv(
    "students.csv",
    header=False,
    schema=schema
)
```

---

## Display Data

```python
df.show()
```

Output:

```text
+------+-------+------+---------+
|RollNo|Section|Name  |City     |
+------+-------+------+---------+
|101   |A      |Ravi  |Hyderabad|
|102   |B      |Kiran |Chennai  |
+------+-------+------+---------+
```

---

# Schema Inference

Sometimes datasets contain hundreds of columns.

Manually defining schema becomes difficult.

Spark can automatically detect schema.

---

## Example

```python
df = spark.read.csv(
    "students.csv",
    inferSchema=True
)
```

---

Spark automatically determines:

```text
Integer
String
Double
Date
etc.
```

---

# Advantage

Less coding effort.

---

# Disadvantage

For large datasets:

```text
Spark must scan data
      ↓
Infer Data Types
      ↓
Extra Time
```

This increases loading time.

---

# Best Practice

### Small Dataset

```python
inferSchema=True
```

Acceptable.

---

### Large Dataset

```python
Define Schema Explicitly
```

Recommended.

---

# Column Names after Infer Schema

When schema is inferred without headers:

```text
_c0
_c1
_c2
_c3
```

Spark generates temporary names.

---

# Renaming Columns

---

## Method 1: withColumnRenamed()

```python
df = df.withColumnRenamed(
    "_c0",
    "RollNo"
)
```

---

Example:

```python
df = df.withColumnRenamed("_c0","RollNo") \
       .withColumnRenamed("_c1","Section") \
       .withColumnRenamed("_c2","Name") \
       .withColumnRenamed("_c3","City")
```

---

# Method 2: toDF()

Provide all names together.

```python
df = df.toDF(
"RollNo",
"Section",
"Name",
"City"
)
```

---

This is simpler and cleaner.

---

# Summary Table

| Method                | Description                      |
| --------------------- | -------------------------------- |
| `RDD.toDF()`          | Create DataFrame from RDD        |
| `createDataFrame()`   | Create DataFrame from Collection |
| `read.csv()`          | Create DataFrame from CSV        |
| `StructType()`        | Define Schema                    |
| `StructField()`       | Define Column                    |
| `inferSchema=True`    | Auto Detect Schema               |
| `withColumnRenamed()` | Rename Column                    |
| `toDF()`              | Rename Multiple Columns          |
