# Apache Spark DataFrames – Structured Notes

---

# Why Do We Need DataFrames?

So far, we have worked with **RDDs (Resilient Distributed Datasets)**.

RDDs are powerful and flexible, but they have some important limitations when working with structured data.

To overcome these limitations, Spark introduced **DataFrames**.

---

# Drawbacks of RDDs

## 1. No Schema Information

RDDs do not understand the structure of data.

Consider the following employee data:

```text
101 Ram HR 50000
102 Riya IT 60000
103 Kiran Finance 70000
```

When stored in an RDD:

```python
[
 ['101','Ram','HR','50000'],
 ['102','Riya','IT','60000']
]
```

Spark only sees:

```text
Collection of Strings
```

It does not know:

* Roll Number → Integer
* Name → String
* Department → String
* Salary → Integer

Everything is treated as raw values.

---

### Problem

Spark cannot automatically:

* Understand column meanings
* Validate data types
* Optimize queries based on schema

---

## 2. No Built-in Query Optimization

Suppose we have two queries:

### Query 1

```python
rdd.join(otherRDD).filter(...)
```

### Query 2

```python
rdd.filter(...).join(otherRDD)
```

Query 2 may be much faster because:

```text
Filter First
      ↓
Less Data
      ↓
Join
```

But Spark RDD API does not know which query is better.

---

### Why?

RDDs are low-level APIs.

Spark only executes what you write.

It cannot intelligently optimize the execution plan.

---

## Summary of RDD Limitations

```text
No Schema

No Column Information

No Data Type Awareness

No Automatic Query Optimization

Complex Syntax

Difficult Output Interpretation
```

---

# What is a DataFrame?

## Definition

A DataFrame is a distributed collection of structured or semi-structured data organized into rows and columns.

It is very similar to:

* Database Tables
* Pandas DataFrames
* Excel Sheets

---

## Visualization

### RDD

```text
['101','Ram','HR','50000']
['102','Riya','IT','60000']
```

---

### DataFrame

| EmpID | Name | Department | Salary |
| ----- | ---- | ---------- | ------ |
| 101   | Ram  | HR         | 50000  |
| 102   | Riya | IT         | 60000  |

Much easier to understand.

---

# DataFrame Structure

A DataFrame consists of:

## Rows

Represent records.

Example:

```text
101 Ram HR 50000
```

---

## Columns

Represent attributes.

```text
EmpID
Name
Department
Salary
```

---

## Schema

Defines:

* Column names
* Data types

Example:

```text
EmpID       Integer
Name        String
Department  String
Salary      Integer
```

---

# Definition

```text
DataFrame
=
Data + Schema
```

---

# Advantages of DataFrames

---

# 1. Structured Format

Data is stored in rows and columns.

Example:

| ID | Name | Salary |
| -- | ---- | ------ |
| 1  | Ram  | 50000  |
| 2  | Riya | 60000  |

Much easier to read and analyze.

---

# 2. Easier Syntax

DataFrame operations are simpler than RDD operations.

---

### Example: Average Score

Suppose we have:

| Name   | Score |
| ------ | ----- |
| Amanda | 20    |
| Amanda | 40    |
| John   | 30    |
| John   | 50    |

---

### Using RDD

```python
rdd.map(...)
   .groupByKey(...)
   .mapValues(...)
```

Complex and difficult to read.

---

### Using DataFrame

```python
df.groupBy("Name").avg("Score")
```

Much easier.

---

# 3. Automatic Query Optimization

Spark understands DataFrame operations.

Because it knows:

```text
Columns
Data Types
Operations
Schema
```

Spark can automatically optimize execution.

---

### Example

```python
df.filter(...)
  .join(...)
```

Spark may internally reorder operations for better performance.

---

# Catalyst Optimizer

Spark uses a special optimization engine called:

```text
Catalyst Optimizer
```

---

## What Does It Do?

It analyzes queries and creates an efficient execution plan.

---

### Without Catalyst

```text
Execute Exactly As Written
```

---

### With Catalyst

```text
Analyze Query
      ↓
Optimize Query
      ↓
Generate Efficient Plan
```

---

## Benefits

✅ Faster execution

✅ Less memory usage

✅ Better resource utilization

---

# 4. Better Performance

DataFrames are generally faster than RDDs because:

```text
Schema Awareness
+
Catalyst Optimizer
+
Execution Optimization
```

---

# 5. Easier Output Interpretation

RDD Output:

```python
[
 ('Amanda',20),
 ('Amanda',40)
]
```

---

DataFrame Output:

| Name   | Avg Score |
| ------ | --------- |
| Amanda | 30        |
| John   | 40        |

Much easier to understand.

---

# RDD vs DataFrame

| Feature            | RDD          | DataFrame  |
| ------------------ | ------------ | ---------- |
| Structure          | Unstructured | Structured |
