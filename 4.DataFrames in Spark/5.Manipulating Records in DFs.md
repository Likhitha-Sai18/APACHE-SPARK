# 1. Sorting Data in Spark

## Why Sorting?

Sorting helps us:

✅ Find top records

✅ Rank data

✅ Analyze trends

✅ Prepare reports

---

# orderBy()

Used to sort records.

---

## Syntax

```python
df.orderBy(columns)
```

---

# Sort by Single Column

Suppose we want to sort by:

```text
Hospital_Code
```

---

## Example

```python
sample_data.orderBy(
    ["Hospital_Code"]
).show()
```

---

## Default Behavior

```text
Ascending Order
```

---

### Output

| Hospital_Code |
| ------------- |
| 1             |
| 2             |
| 5             |
| 8             |

---

# Descending Order

Use:

```python
ascending=False
```

---

## Example

```python
sample_data.orderBy(
    ["Hospital_Code"],
    ascending=False
).show()
```

---

### Output

| Hospital_Code |
| ------------- |
| 32            |
| 26            |
| 20            |
| 8             |

---

# Multiple Column Sorting

---

Suppose:

```text
1st Sort → Hospital_Code (Ascending)

2nd Sort → Visitors_with_Patient (Descending)
```

---

## Example

```python
sample_data.orderBy(
    ["Hospital_Code",
     "Visitors_with_Patient"],

    ascending=[True, False]
)
```

---

# Visualization

```text
Hospital_Code ↑

Within Same Hospital

Visitors_with_Patient ↓
```

---

# Summary

| Function        | Purpose    |
| --------------- | ---------- |
| orderBy()       | Sort Data  |
| ascending=True  | Ascending  |
| ascending=False | Descending |

---

# 3. Aggregation Functions

After grouping, we apply aggregation.

---

Common Aggregations:

```python
avg()
sum()
count()
min()
max()
countDistinct()
```

---

# Average Visitors per Hospital

---

## Example

```python
from pyspark.sql.functions import avg
```

---

```python
sample_data.groupBy(
    "Hospital_Code"
).agg(
    avg("Visitors_with_Patient")
).show()
```

---

### Output

| Hospital_Code | Avg Visitors |
| ------------- | ------------ |
| 8             | 3.4          |
| 10            | 2.8          |

---

# Renaming Aggregated Columns

By default:

```text
avg(Visitors_with_Patient)
```

is not a friendly name.

---

# alias()

Used to rename aggregation columns.

---

## Example

```python
avg(
"Visitors_with_Patient"
).alias(
"Average_Visitors"
)
```

---

### Complete Query

```python
sample_data.groupBy(
"Hospital_Code"
).agg(
avg(
"Visitors_with_Patient"
).alias(
"Average_Visitors"
)
)
```

---

# Count Unique Patients

---

Suppose we need:

```text
Unique Patients
per Department
```

---

## Import

```python
from pyspark.sql.functions import countDistinct
```

---

## Query

```python
sample_data.groupBy(
"Department"
).agg(
countDistinct(
"PatientID"
).alias(
"Unique_Patients"
)
)
```

---

### Output

| Department | Unique_Patients |
| ---------- | --------------- |
| Radiology  | 300             |
| Surgery    | 420             |

---

# Multiple Aggregations

Spark allows multiple aggregations together.

---

## Example

Need:

```text
Unique Patients

AND

Average Visitors
```

per Department.

---

```python
sample_data.groupBy(
"Department"
).agg(

countDistinct(
"PatientID"
).alias(
"Unique_Patients"
),

avg(
"Visitors_with_Patient"
).alias(
"Avg_Visitors"
)

)
```

---

### Output

| Department | Unique_Patients | Avg_Visitors |
| ---------- | --------------- | ------------ |
| Surgery    | 450             | 3.2          |
| Radiology  | 300             | 2.8          |

---

# Aggregation Flow

```text
DataFrame

      ↓

groupBy()

      ↓

agg()

      ↓

avg()
count()
sum()
max()
min()
```

# 4. Joins in Spark

---

# Why Joins?

Real-world data is usually stored across multiple tables.

Example:

```text
Patient Data

+

Case URL Data

+

Hospital Data
```

Need to combine them.

---

# Example Tables

---

## Healthcare Data

| Case_ID | Hospital |
| ------- | -------- |
| 1       | A        |
| 2       | B        |
| 3       | C        |

---

## Case URL Data

| Case_ID | URL  |
| ------- | ---- |
| 1       | url1 |
| 2       | url2 |
| 4       | url4 |

---

# Join Syntax

```python
df1.join(
    df2,
    join_column,
    join_type
)
```

---

# Inner Join

---

# Definition

Returns:

```text
Only Matching Records
```

---

## Example

```python
sample_data.join(
case_url_df,
"Case_ID",
"inner"
)
```

---

### Result

| Case_ID |
| ------- |
| 1       |
| 2       |

Case 3 and 4 removed.

---

# Visualization

```text
Table A      Table B

1 2 3        1 2 4

INNER JOIN

1 2
```

---

# Right Join

---

# Definition

Returns:

```text
All Records from Right Table

+
Matching Records from Left Table
```

---

## Example

```python
sample_data.join(
case_url_df,
"Case_ID",
"right"
)
```

---

### Result

| Case_ID |
| ------- |
| 1       |
| 2       |
| 4       |

---

# Visualization

```text
Left

1 2 3

Right

1 2 4

RIGHT JOIN

1 2 4
```

---

# Other Joins You Should Practice

---

## Left Join

```python
join(
df2,
"id",
"left"
)
```

Returns:

```text
All Left Records
+
Matching Right Records
```

---

## Full Outer Join

```python
join(
df2,
"id",
"outer"
)
```

Returns:

```text
All Records
from both tables
```

---

# Join Summary

| Join Type  | Returns                   |
| ---------- | ------------------------- |
| Inner      | Matching Records          |
| Left       | All Left + Matching Right |
| Right      | All Right + Matching Left |
| Full Outer | All Records               |

# DataFrame vs RDD

Now comes an important interview topic.

---

# Why DataFrames were Introduced?

RDDs had limitations.

---

# Limitation 1: No Schema

RDD

```python
[
("Ravi",20),
("Kiran",30)
]
```

Spark does not know:

```text
Name Column

Age Column
```

---

DataFrame

```text
Name      Age
Ravi      20
Kiran     30
```

Schema available.

---

# Limitation 2: No Query Optimization

RDD:

```python
rdd.join().filter()
```

Spark cannot optimize.

---

DataFrame:

```python
df.join().filter()
```

Spark can optimize automatically.

---

# Catalyst Optimizer

DataFrames use:

```text
Catalyst Optimizer
```

---

Catalyst:

```text
Analyzes Query

Optimizes Query

Improves Performance
```
