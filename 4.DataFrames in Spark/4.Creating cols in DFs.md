# Apache Spark DataFrames – Creating New Columns (Structured Notes)

> Feature Engineering is one of the most important tasks in Data Engineering and Data Science. Spark provides multiple ways to create new columns from existing data.

---

# Why Create New Columns?

Creating new columns helps us:

✅ Derive hidden insights

✅ Transform raw data into meaningful features

✅ Prepare data for Machine Learning

✅ Simplify analysis

---

## Example

Original Data

| Ward_Type | Visitors_with_Patient |
| --------- | --------------------- |
| Q         | 6                     |
| R         | 3                     |
| S         | 2                     |

Create New Feature:

| Ward_Type | Visitors_with_Patient | Ward_Flag |
| --------- | --------------------- | --------- |
| Q         | 6                     | 2         |
| R         | 3                     | 1         |
| S         | 2                     | 1         |

This process is called **Feature Engineering**.

---

# Methods to Create New Columns

Spark provides 3 major approaches:

```text
1. expr() Function

2. when() / otherwise()

3. User Defined Functions (UDF)
```

---

# Import Required Libraries

```python
from pyspark.sql import SparkSession

from pyspark.sql.functions import (
    expr,
    when,
    udf
)

from pyspark.sql.types import IntegerType
```

---

# Create Spark Session

```python
spark = SparkSession.builder \
       .appName("FeatureEngineering") \
       .getOrCreate()
```

---

# Method 1: Using expr()

---

# What is expr()?

`expr()` allows us to write SQL-like expressions inside Spark.

---

## Syntax

```python
expr("expression")
```

---

# Example 1: Add Static Column

Suppose we want every row to contain:

```text
Country = India
```

---

## Create Expression

```python
country_value = expr("'India'")
```

---

## Create New Column

```python
updated_df = sample_data.withColumn(
    "Country",
    country_value
)
```

---

## Output

| Hospital_Code | Department | Country |
| ------------- | ---------- | ------- |
| 8             | Radiology  | India   |
| 10            | Surgery    | India   |

---

# Visualization

```text
Original DataFrame

Hospital_Code
Department

        ↓

withColumn()

        ↓

Hospital_Code
Department
Country
```

---

# Example 2: Conditional Column using expr()

---

Condition:

```text
If Ward_Type = 'Q'
      → 2

Else
      → 0
```

---

## Expression

```python
filter_expression = expr(
"""
CASE
WHEN Ward_Type='Q'
THEN 2
ELSE 0
END
"""
)
```

---

## Create Column

```python
updated_df = sample_data.withColumn(
    "Ward_Flag",
    filter_expression
)
```

---

## Output

| Ward_Type | Ward_Flag |
| --------- | --------- |
| Q         | 2         |
| R         | 0         |
| S         | 0         |

---

# Method 2: Using when() and otherwise()

---

# Why Use It?

More readable than expr()

Most commonly used in real projects.

---

## Syntax

```python
when(condition, value) \
.otherwise(value)
```

---

# Example 1

Condition:

```text
Ward_Type = Q → 2
Otherwise → 0
```

---

```python
updated_df = sample_data.withColumn(
    "Ward_Flag",
    when(
        sample_data.Ward_Type == "Q",
        2
    ).otherwise(0)
)
```

---

## Output

| Ward_Type | Ward_Flag |
| --------- | --------- |
| Q         | 2         |
| R         | 0         |
| S         | 0         |

---

# Multiple Conditions

---

Condition:

```text
If Ward_Type = Q
      → 2

Else If Visitors_with_Patient < 4
      → 1

Else
      → 0
```

---

## Implementation

```python
updated_df = sample_data.withColumn(
    "Ward_Visitor_Flag",

    when(
        sample_data.Ward_Type == "Q",
        2
    )

    .when(
        sample_data.Visitors_with_Patient < 4,
        1
    )

    .otherwise(0)
)
```

---

## Output

| Ward_Type | Visitors_with_Patient | Ward_Visitor_Flag |
| --------- | --------------------- | ----------------- |
| Q         | 7                     | 2                 |
| R         | 2                     | 1                 |
| S         | 6                     | 0                 |

---

# Execution Flow

```text
Check Ward_Type

       ↓

Ward_Type = Q ?

Yes → 2

No
 ↓

Visitors < 4 ?

Yes → 1

No
 ↓

0
```

---

# Method 3: User Defined Functions (UDF)

---

# What is UDF?

UDF = User Defined Function

Allows us to write custom Python functions and apply them on DataFrames.

---

# When to Use UDF?

Use UDF when:

✅ Logic is complex

✅ Built-in Spark functions cannot solve it

✅ Business rules are complicated

---

# Example

Condition:

```text
Ward_Type = Q
      → 2

Visitors < 4
      → 1

Else
      → 0
```

---

## Step 1: Create Python Function

```python
def encodeWardVisitor(
    ward,
    visitor
):

    if ward == "Q":
        return 2

    elif visitor < 4:
        return 1

    else:
        return 0
```

---

## Step 2: Convert to UDF

```python
function_udf = udf(
    encodeWardVisitor,
    IntegerType()
)
```

---

# Why IntegerType()?

Because our function returns:

```text
0
1
2
```

which are integers.

---

## Step 3: Create New Column

```python
updated_df = sample_data.withColumn(

    "Ward_Visitor_Flag",

    function_udf(
        sample_data.Ward_Type,
        sample_data.Visitors_with_Patient
    )
)
```

---

## Output

| Ward_Type | Visitors_with_Patient | Ward_Visitor_Flag |
| --------- | --------------------- | ----------------- |
| Q         | 6                     | 2                 |
| R         | 2                     | 1                 |
| S         | 5                     | 0                 |

---

# Comparison of All Methods

| Method | Best For              | Performance |
| ------ | --------------------- | ----------- |
| expr() | SQL-style expressions | Fast        |
| when() | Conditional logic     | Fast        |
| UDF    | Custom logic          | Slower      |

---

# Important Interview Point

---

## Why are UDFs slower?

Spark understands:

```python
when()
expr()
```

and optimizes them.

---

Spark cannot understand:

```python
custom Python code
```

inside UDFs.

Therefore:

```text
UDF
 ↓

Serialization

 ↓

Python Execution

 ↓

Deserialization

 ↓

Slower
```

---

# Real-World Recommendation

Always follow:

```text
Built-in Functions
       ↓

when()

       ↓

expr()

       ↓

UDF (Last Option)
```

---

# Most Important Function: withColumn()

All three approaches ultimately use:

```python
withColumn()
```

---

## Syntax

```python
df.withColumn(
    column_name,
    expression
)
```

---

# Uses of withColumn()

---

### Create New Column

```python
df.withColumn(
    "Country",
    lit("India")
)
```

---

### Modify Existing Column

```python
df.withColumn(
    "Age",
    col("Age").cast("int")
)
```

---

### Replace Column Values

```python
df.withColumn(
    "Salary",
    col("Salary") * 2
)
```

---

# Summary Table

| Function      | Purpose              |
| ------------- | -------------------- |
| withColumn()  | Create/Modify Column |
| expr()        | SQL Expression       |
| when()        | Conditional Logic    |
| otherwise()   | Default Value        |
| udf()         | Custom Function      |
| IntegerType() | Return Integer       |
