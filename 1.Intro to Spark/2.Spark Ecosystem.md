# Apache Spark Ecosystem – Structured Notes

## Introduction

The **Apache Spark Ecosystem** is a collection of libraries built on top of **Spark Core** that enable Spark to perform various types of data processing tasks such as:

* Batch Processing
* Real-Time Processing
* SQL Analytics
* Machine Learning
* Graph Processing

These components work together to provide a unified platform for Big Data analytics.

---

# Apache Spark Ecosystem Components

```text
                    Apache Spark
                          │
 ┌──────────┬──────────┬──────────┬──────────┬──────────┐
 │          │          │          │          │
Spark Core Spark SQL Streaming  MLlib    GraphX
```

The Spark ecosystem consists of:

1. Spark Core
2. Spark SQL
3. Spark Streaming
4. MLlib
5. GraphX

---

# 1. Spark Core

## Definition

Spark Core is the **foundation and heart of Apache Spark**. All other Spark components are built on top of Spark Core.

---

## Functions of Spark Core

### Memory Management

* Stores and manages data in memory (RAM)
* Enables faster processing

### Task Scheduling

* Assigns tasks to worker nodes
* Manages execution order

### Job Execution

Converts user applications into:

```text
Application
    ↓
Jobs
    ↓
Stages
    ↓
Tasks
```

### Fault Recovery

* Handles node failures
* Recovers lost data using lineage information

### Resource Management

Manages:

* CPU
* Memory
* Cluster Resources

---

## Importance

Every Spark component depends on Spark Core for:

* Execution
* Scheduling
* Memory Management
* Fault Tolerance

---

# 2. Spark SQL

## Definition

Spark SQL is a module used to process **structured and semi-structured data** using SQL queries.

---

## Data Types Supported

### Structured Data

Data with a fixed schema.

Example:

| ID | Name | Salary |
| -- | ---- | ------ |
| 1  | John | 50000  |

---

### Semi-Structured Data

Data without a fixed schema.

Examples:

* JSON
* XML
* CSV

---

## Features

### SQL Query Support

Example:

```sql
SELECT * FROM employees
WHERE salary > 50000;
```

### DataFrames

A DataFrame is:

> A distributed collection of data organized into rows and columns.

Example:

```python
df.show()
```

### Hive Integration

Spark SQL can work directly with:

* Hive Tables
* Hive Metastore
* Hive Queries

---

## Advantages

* Easy for SQL users
* Faster query execution
* Supports multiple file formats
* Simplifies data analysis

---

# 3. Spark Streaming

## Definition

Spark Streaming is used for processing real-time data streams.

---

## Need for Streaming

Traditional batch processing:

```text
Collect Data
      ↓
Store Data
      ↓
Process Later
```

Not suitable for applications requiring immediate response.

---

## Working

```text
Incoming Data
      ↓
Mini Batches
      ↓
Continuous Processing
      ↓
Results
```

---

## Features

### Real-Time Processing

Processes data as it arrives.

Examples:

* Fraud Detection
* Live Monitoring
* Social Media Analytics

### Structured Streaming

Allows processing of:

* Structured Data
* Streaming Data

simultaneously.

### Fast Recovery

Recovers automatically from failures.

### Multiple Data Sources

Can receive data from:

* Kafka
* Flume
* HDFS
* IoT Devices
* Sockets

---

## Example

Credit Card Fraud Detection:

```text
Transaction
     ↓
Spark Streaming
     ↓
Fraud Analysis
     ↓
Immediate Action
```

# 5. GraphX

## Definition

GraphX is Spark's graph processing library used to analyze relationships between data.

---

## Graph Basics

A graph consists of:

### Vertices (Nodes)

Represent entities.

Examples:

* Users
* Products
* Locations

### Edges (Links)

Represent relationships.

Example:

```text
User A ─── Friend ─── User B
```

---

## Graph Representation

```text
     A
    / \
   B---C
```

Where:

* A, B, C → Vertices
* Lines → Edges

---

## Applications

### Social Networks

Analyze:

* Friend Connections
* User Relationships

### Recommendation Systems

Identify similar users and products.

### Network Analysis

Used in:

* Communication Networks
* Transportation Systems

---

## Limitation

* Primarily available through Scala APIs.
* Less commonly used than Spark SQL and MLlib.
