## Definition

Apache Spark is an **open-source distributed data processing engine** designed for processing **large-scale datasets quickly and efficiently** across multiple computers (a cluster).

It was developed to overcome the speed limitations of Hadoop MapReduce by performing much of its computation **in memory (RAM)** rather than repeatedly reading from and writing to disk.

---

## Simple Explanation

Imagine you have:

* 1 GB of data → Your laptop can handle it.
* 100 TB of data → A single computer cannot process it efficiently.

Spark solves this by:

1. Splitting the data into smaller pieces.
2. Distributing those pieces across many machines.
3. Processing them simultaneously (parallel processing).
4. Combining the results.

This allows Spark to analyze massive datasets much faster than traditional methods.

---

## Why Was Spark Created?

### Hadoop MapReduce Problem

```text
Read Data
    ↓
Process
    ↓
Write to Disk
    ↓
Read Again
    ↓
Process Again
```

Too many disk operations make processing slow.

### Spark Solution

```text
Read Data
    ↓
Store in Memory (RAM)
    ↓
Process Multiple Times
    ↓
Return Result
```

Less disk I/O = Much faster execution.

# Apache Spark – Structured Notes

## 1. What is Apache Spark?

**Apache Spark** is an **open-source distributed data processing framework** used for processing and analyzing **large volumes of data** quickly.

It was developed to overcome the performance limitations of Hadoop MapReduce by using **in-memory computation**, making data processing much faster.

### Definition

> Apache Spark is a fast, open-source, distributed computing engine designed for large-scale data processing, analytics, machine learning, and real-time data streaming.

---

# 2. Why Was Spark Introduced?

### Problem with Hadoop MapReduce

In Hadoop MapReduce:

```text
Read Data
    ↓
Process
    ↓
Write to Disk
    ↓
Read Again
    ↓
Process Again
```

For iterative tasks such as Machine Learning:

* Data is repeatedly written to disk.
* Disk I/O operations are slow.
* Performance decreases significantly.

### Spark Solution

Spark stores intermediate results in **RAM (memory)** instead of repeatedly writing them to disk.

```text
Read Data
    ↓
Store in Memory
    ↓
Process Multiple Times
    ↓
Final Output
```

### Result

* Reduced Disk I/O
* Faster Execution
* Better Performance

---

# 3. Key Characteristics of Spark

### Open Source

Spark is freely available and maintained by the Apache Software Foundation.

### Distributed Processing

Data is divided into smaller chunks and processed across multiple machines simultaneously.

### In-Memory Computing

Stores intermediate results in RAM for faster computation.

### Fault Tolerance

Spark automatically recovers lost data using lineage information.

### Scalability

Can scale from a single machine to thousands of nodes.

---

## 4.3 Unified Analytics Engine

Spark combines multiple data processing capabilities into a single framework.

| Component       | Purpose                         |
| --------------- | ------------------------------- |
| Spark Core      | Basic data processing           |
| Spark SQL       | SQL queries and structured data |
| MLlib           | Machine Learning                |
| Spark Streaming | Real-time processing            |
| GraphX          | Graph analytics                 |

### Benefit

Developers do not need multiple tools for different tasks.

---

## 4.4 In-Memory Computation

This is Spark's most important feature.

### Hadoop MapReduce

```text
Memory → Disk → Memory → Disk
```

### Spark

```text
Memory → Process → Process → Process
```

### Benefits

* Reduced Disk I/O
* Faster Processing
* Better Performance for iterative algorithms

### Drawback

Requires more RAM.

---

## 4.5 Batch and Real-Time Processing

### Hadoop

Supports only Batch Processing.

Example:

* Monthly reports
* Daily transaction summaries

### Spark

Supports:

* Batch Processing
* Near Real-Time Processing

Example:

* Fraud Detection
* Live Analytics
* Stock Market Monitoring

# 5. Apache Spark Architecture

## Components

```text
Driver Program
       |
       |
--------------------------------
|              |              |
Executor 1   Executor 2   Executor 3
|              |              |
Tasks         Tasks         Tasks
```

---

## Driver Program

Responsibilities:

* Creates Spark Context
* Divides work into tasks
* Schedules jobs
* Monitors execution

---

## Executors

Responsibilities:

* Execute tasks
* Store data in memory
* Return results to driver

---

## Cluster Manager

Responsible for resource management.

Examples:

* YARN
* Kubernetes
* Mesos
* Spark Standalone

# 7. Spark vs Hadoop MapReduce

| Feature                  | Hadoop MapReduce | Spark     |
| ------------------------ | ---------------- | --------- |
| Processing Speed         | Slow             | Fast      |
| In-Memory Processing     | No               | Yes       |
| Batch Processing         | Yes              | Yes       |
| Real-Time Processing     | No               | Yes       |
| Machine Learning Support | Limited          | Excellent |

| RAM Requirement          | Low              | High      |
| Cost                     | Lower            | Higher    |

Needs large RAM resources.
