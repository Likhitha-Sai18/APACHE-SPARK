# Step 1: Import SparkSession

SparkSession is the entry point to Spark SQL.

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder.appName("SparkSQL").getOrCreate()
```

### Why?

SparkSession provides:

* SQL engine
* DataFrame API
* File reading
* Temporary tables

---

# Step 2: Dataset

The example uses **Census Data**.

Sample columns:

* age
* workclass
* education
* occupation
* marital_status
* income
* fnlwgt
* hours_per_week

Each row represents one person.

# Step 3: Define Schema

Instead of Spark guessing data types, define them manually.

```python
StructType([
StructField("age", IntegerType()),
StructField("workclass", StringType()),
...
])
```

### Benefits

* Faster reading
* Correct data types
* Avoids inference errors

---

# Step 4: Read Data into DataFrame

```python
df = spark.read.csv(
"path",
schema=schema,
header=True
)
```

Now Spark creates a DataFrame.

Think of it as:

```text
CSV
 │
 ▼
DataFrame
```

---

# Step 5: Register DataFrame as Temporary View

SQL cannot directly query a DataFrame.

So register it as a temporary SQL table.

```python
df.createOrReplaceTempView("census")
```

Now Spark creates:

```text
DataFrame
      │
      ▼
Temporary View

Name = census
```

Now SQL queries become possible.

---

# Step 6: Run SQL Query

```python
spark.sql("""
SELECT *
FROM census
""")
```

Spark executes SQL on the DataFrame.

---

# Step 7: Select Specific Columns

Instead of selecting everything:

```sql
SELECT age,
occupation
FROM census
```

Only required columns are returned.

Benefits:

* Less memory
* Faster execution

---

# Step 8: Create New Column

Example

```sql
SELECT *,
fnlwgt % 1000 AS new_column
FROM census
```

Creates a new calculated column.

Important:

This does **NOT** modify the original DataFrame.

Store it in another DataFrame.

```python
df_model = spark.sql(...)
```

---

# Step 9: Register New DataFrame Again

To run SQL again:

```python
df_model.createOrReplaceTempView("census_model")
```

Now SQL queries must use:

```sql
FROM census_model
```

instead of

```sql
FROM census
```

# Step 10: Aggregate Functions

Spark SQL supports SQL functions just like MySQL.

Example:

* SUM()
* AVG()
* COUNT()
* ROUND()
* LTRIM()
* MAX()
* MIN()

Example:

```sql
SELECT
occupation,
AVG(age)
FROM census_model
GROUP BY occupation
```

# Step 13: Multiple Functions Together

Spark SQL allows many functions in one query.

Example

```sql
SELECT

occupation,

ROUND(
AVG(age),2
),

SUM(income),

COUNT(*)
```

Exactly like MySQL.

# Step 14: Convert SQL Table to DataFrame

Spark provides

```python

```

Now

```text
Temporary View

↓

```

No need to read CSV again.

# Step 15: Why Convert to Pandas?

Spark DataFrames are distributed.

Libraries like

* Matplotlib
* Seaborn

need Pandas.

So convert:

```python
pdf = spark_df.toPandas()
```

---

## Important

Never convert huge Spark DataFrames.

Instead:

```text
Large Data

↓

Aggregate in Spark

↓

Small Result

↓

Convert to Pandas
```

This saves memory.

# Best Practices

✔ Define schema manually.

✔ Register DataFrames as temporary views before SQL.

✔ Save modified results into a new DataFrame.

✔ Register the new DataFrame if more SQL is needed.

✔ Aggregate data before converting to Pandas.

✔ Convert only small datasets to Pandas.

# Interview Questions

### 1. Why use `createOrReplaceTempView()`?

It registers a DataFrame as a temporary SQL table so SQL queries can be executed on it.

---

### 2. Can SQL directly query a DataFrame?

No. First, create a temporary view using `createOrReplaceTempView()`.

---

### 3. Why define a schema manually?

* Faster reading
* Correct data types
* Avoids incorrect type inference

---

### 4. Why convert Spark DataFrame to Pandas?

Visualization libraries like Matplotlib work with Pandas DataFrames.

---

### 5. Why shouldn't large Spark DataFrames be converted to Pandas?

Because Pandas stores all data in the driver's memory, which can cause memory overflow. It's better to aggregate the data in Spark first, then convert only the smaller result.

# Quick Revision

* **SparkSession** → Entry point for Spark SQL.
* **Schema** → Defines column names and data types.
* **DataFrame** → Distributed table-like data structure.
* **Temporary View** → Makes a DataFrame queryable using SQL.
* **spark.sql()** → Executes SQL queries.
* **New DataFrame** → Stores query results; original DataFrame remains unchanged.
* **spark.table()** → Converts a temporary view back into a DataFrame.
* **toPandas()** → Converts a small Spark DataFrame into a Pandas DataFrame.
* **Matplotlib** → Used to visualize the final aggregated results.
