# Apache Spark Pair RDD Transformations and Actions – Structured Notes

---

# Introduction

Pair RDDs are one of the most powerful features of Spark because they allow processing data in **Key-Value Pair format**.

Example:

```text
(Student, Score)

(A, 10)
(B, 20)
(A, 30)
(B, 40)
```

Where:

* **Key** = Student Name
* **Value** = Score

Pair RDDs support special transformations and actions that normal RDDs do not.

---

# Sample Pair RDD

Let's assume we have the following Pair RDD:

```python
data = [
('A',10),
('B',20),
('A',30),
('B',40),
('A',50)
]

pairRDD = sc.parallelize(data)
```

Output:

```text
(A,10)
(B,20)
(A,30)
(B,40)
(A,50)
```

---

# Pair RDD Transformations

## What are Pair RDD Transformations?

These are transformations specifically designed for Key-Value Pair data.

Common transformations:

```text
1. groupByKey()
2. reduceByKey()
3. sortByKey()
4. join()
```

---

# 1. groupByKey()

## Definition

`groupByKey()` groups all values belonging to the same key.

---

# Example

### Input Pair RDD

```text
(A,10)
(B,20)
(A,30)
(B,40)
(A,50)
```

---

## Code

```python
pairRDD.groupByKey().collect()
```

---

## Output

```text
A → [10,30,50]

B → [20,40]
```

---

# Working of groupByKey()

### Step 1

Original Data

```text
Partition 1

(A,10)
(B,20)

Partition 2

(A,30)
(B,40)

Partition 3

(A,50)
```

---

### Step 2

Shuffle Data Across Network

```text
A → [10,30,50]

B → [20,40]
```

All values for each key are moved into the same partition.

---

### Step 3

Return Iterable Values

```text
(A,[10,30,50])

(B,[20,40])
```

---

# Calculate Total Marks Using groupByKey()

```python
pairRDD.groupByKey() \
       .mapValues(sum) \
       .collect()
```

Output:

```text
(A,90)

(B,60)
```

---

# Why groupByKey() is Not Recommended?

Because it performs:

```text
Full Data Shuffle
```

before aggregation.

---

## Problem

Suppose:

```text
(A,10)

(A,20)

(A,30)

(A,40)

(A,50)
```

All records must travel through the network.

```text
Huge Network Traffic
```

For Big Data:

❌ Slow

❌ High Memory Usage

❌ Expensive

---

# 2. reduceByKey()

## Definition

`reduceByKey()` combines values within each partition before shuffling.

---

# Why is reduceByKey Better?

Instead of moving all values:

```text
10
20
30
40
50
```

Spark performs local aggregation first.

---

# Example

Input:

```text
(A,10)

(B,20)

(A,30)

(B,40)

(A,50)
```

---

## Code

```python
pairRDD.reduceByKey(
lambda x,y:x+y
).collect()
```

---

## Output

```text
(A,90)

(B,60)
```

---

# Working of reduceByKey()

## Step 1: Local Aggregation

Partition 1

```text
(A,10)
(A,30)

↓

(A,40)
```

Partition 2

```text
(A,50)

↓

(A,50)
```

---

## Step 2: Shuffle Reduced Data

Instead of shuffling:

```text
10
30
50
```

Spark shuffles:

```text
40
50
```

Much smaller data.

---

## Step 3: Final Aggregation

```text
40 + 50

↓

90
```

Final Output:

```text
(A,90)
```

---

# groupByKey() vs reduceByKey()

| Feature           | groupByKey() | reduceByKey() |
| ----------------- | ------------ | ------------- |
| Groups Values     | Yes          | No            |
| Aggregates Values | No           | Yes           |
| Network Shuffle   | High         | Low           |
| Performance       | Slow         | Fast          |
| Recommended       | No           | Yes           |

---

# Interview Question

### Which is better?

```text
reduceByKey()
```

because it minimizes network traffic.

---

# Visual Comparison

## groupByKey()

```text
All Data
      ↓
Shuffle
      ↓
Group
      ↓
Aggregate
```

---

## reduceByKey()

```text
Aggregate
      ↓
Shuffle
      ↓
Aggregate Again
```

Less data movement.

# Pair RDD Actions

Actions trigger execution and return results.

# 1. countByKey()

## Definition

Counts number of values associated with each key.

---

# Example

Input:

```text
(A,10)

(B,20)

(A,30)

(B,40)

(A,50)
```

---

## Code

```python
pairRDD.countByKey()
```

---

## Output

```text
{
 'A':3,
 'B':2
}
```

---

# Working

```text
A → 3 Values

B → 2 Values
```

---

# Real-Life Example

Employee Departments:

```text
(HR,Ram)

(HR,Riya)

(IT,Amit)

(HR,John)
```

Output:

```text
HR → 3

IT → 1
```

---

# 2. lookup()

## Definition

Returns all values associated with a given key.

---

# Example

Input:

```text
(A,10)

(B,20)

(A,30)

(B,40)

(A,50)
```

---

## Code

```python
pairRDD.lookup('A')
```

---

## Output

```text
[10,30,50]
```

---

# Another Example

```python
pairRDD.lookup('B')
```

Output:

```text
[20,40]
```

---

# Working

```text
Key = A

↓

Find All Records

↓

Return Values

↓

[10,30,50]
```

---

# Complete Example

```python
from pyspark import SparkContext

sc = SparkContext()

data = [
('A',10),
('B',20),
('A',30),
('B',40),
('A',50)
]

pairRDD = sc.parallelize(data)
```

---

```text
 ('A',90),
```


# Memory Trick

### GRCL Rule

**G → groupByKey()**

**R → reduceByKey()**

**C → countByKey()**

**L → lookup()**

Remember:

> **groupByKey groups, reduceByKey reduces, countByKey counts, lookup retrieves.**

These four operations are among the most frequently asked Pair RDD interview concepts and are heavily used in real-world Spark data processing. 🚀
