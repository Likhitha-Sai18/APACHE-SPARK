# Apache Spark RDD Operations – Complete Structured Notes with Examples

---

# Introduction

After creating an **RDD (Resilient Distributed Dataset)**, we perform operations on it to process data.

RDD operations are classified into two categories:

```text
RDD Operations
      │
 ┌────┴────┐
 │         │
Transformations   Actions
```

---

# 1. Transformations

## Definition

Transformations are operations that create a **new RDD from an existing RDD**.

Since RDDs are **immutable**, any modification results in a new RDD.

### Important Characteristics

✔ Create New RDD

✔ Lazy Evaluation

✔ Do Not Execute Immediately

✔ Can Be Chained Together

---

## Example

### Original RDD

```python
rdd = sc.parallelize([1,2,3,4])
```

### Apply Transformation

```python
rdd1 = rdd.map(lambda x: x*2)
```

### Result (Not Executed Yet)

```text
RDD = [1,2,3,4]

RDD1 = [2,4,6,8]
```

Spark only creates an execution plan.

---

# 2. Actions

## Definition

Actions trigger execution and return results to the Driver Program.

### Characteristics

✔ Execute Transformations

✔ Return Results

✔ End the Lazy Evaluation Chain

---

## Example

```python
rdd1.collect()
```

### Output

```text
[2,4,6,8]
```

Now Spark executes all pending transformations.

---

# Lazy Evaluation

## Definition

Spark delays execution until an action is called.

---

## Example

```python
rdd = sc.parallelize([1,2,3,4])

rdd1 = rdd.map(lambda x:x*2)

rdd2 = rdd1.filter(lambda x:x>4)
```

### At This Point

```text
No Execution Happens
```

Spark only stores:

```text
RDD
 ↓
Map
 ↓
Filter
```

---

## Action Trigger

```python
rdd2.collect()
```

Now Spark executes:

```text
RDD Creation
      ↓
Map
      ↓
Filter
      ↓
Collect
```

# Common Transformation Functions

---

# 1. map()

## Definition

Applies a function to every element of an RDD.

---

## Syntax

```python
rdd.map(function)
```

---

## Example 1: Multiply Numbers

```python
rdd = sc.parallelize([1,2,3,4])

result = rdd.map(lambda x:x*10)
```

### Output

```text
[10,20,30,40]
```

---

## Example 2: Tokenize Employee Records

### Before

```text
"John HR India"
```

### Code

```python
tokenized = analytics.map(
lambda x:x.split()
)
```

### Output

```text
[
 ['John','HR','India'],
 ['Sarah','Finance','Australia'],
 ['Tarek','HR','Dubai']
]
```

---

## Working

```text
Original Record
      ↓
John HR India
      ↓
split()
      ↓
['John','HR','India']
```

Applied to every record.

---

# 2. flatMap()

## Definition

Similar to map() but flattens the output.

---

## Example

```python
rdd = sc.parallelize([
"Big Data",
"Apache Spark"
])
```

---

### Using map()

```python
rdd.map(lambda x:x.split()).collect()
```

### Output

```text
[
 ['Big','Data'],
 ['Apache','Spark']
]
```

---

### Using flatMap()

```python
rdd.flatMap(lambda x:x.split()).collect()
```

### Output

```text
[
 'Big',
 'Data',
 'Apache',
 'Spark'
]
```

---

## Difference

| map()         | flatMap()        |
| ------------- | ---------------- |
| Nested Output | Flattened Output |
| List of Lists | Single List      |

---

# 3. filter()

## Definition

Returns only records satisfying a condition.

---

## Syntax

```python
rdd.filter(condition)
```

---

## Example 1: Even Numbers

```python
rdd = sc.parallelize([1,2,3,4,5,6])

rdd.filter(
lambda x:x%2==0
).collect()
```

### Output

```text
[2,4,6]
```

---

## Example 2: Employees from India

```python
india_emp = tokenized.filter(
lambda x:x[2]=="India"
)
```

### Output

```text
[
 ['John','HR','India'],
 ['Riya','HR','India']
]
```

---

# 4. distinct()

## Definition

Returns only unique values.

---

## Example

```python
places = tokenized.map(
lambda x:x[2]
)
```

### Output

```text
India
Australia
Dubai
Scotland
India
Dubai
```

---

### Apply distinct()

```python
places.distinct().collect()
```

### Output

```text
[
 'India',
 'Australia',
 'Dubai',
 'Scotland'
]
```

---

# Common Action Functions

---

# 1. collect()

## Definition

Returns all elements from an RDD.

---

## Example

```python
rdd.collect()
```

### Output

```text
[1,2,3,4]
```

---

## Warning

For large datasets:

```text
Entire Dataset
      ↓
Driver Memory
```

May cause OutOfMemory errors.

---

# 2. take()

## Definition

Returns first N elements.

---

## Syntax

```python
rdd.take(n)
```

---

## Example

```python
rdd.take(3)
```

### Output

```text
[1,2,3]
```

---

## Why Better Than collect()?

Only retrieves required data.

```text
take(5)
     ↓
Only 5 Records
```

---

# 3. count()

## Definition

Returns number of elements in RDD.

---

## Example

```python
rdd.count()
```

### Output

```text
6
```

---

     ↓

# Summary Table

## Transformations

| Transformation | Purpose                           |
| -------------- | --------------------------------- |
| map()          | Apply function to every record    |
| flatMap()      | Apply function and flatten output |
| filter()       | Select records matching condition |
| distinct()     | Remove duplicate values           |

---

## Actions

| Action    | Purpose                 |
| --------- | ----------------------- |
| collect() | Return all elements     |
| take(n)   | Return first N elements |
| count()   | Count records           |

---

# Interview Questions

### Q1. What is the difference between Transformations and Actions?

| Transformation  | Action               |
| --------------- | -------------------- |
| Creates New RDD | Returns Result       |
| Lazy            | Executes Immediately |
| No Output       | Produces Output      |

---

### Q2. Why are transformations called lazy?

Because Spark does not execute them until an action is called.

---

### Q3. Difference between map() and flatMap()?

```text
map()      → Nested Output

flatMap()  → Flattened Output
```

---

### Q4. Why is take() preferred over collect()?

Because it retrieves only required records and uses less memory.

---

### Q5. What does distinct() do?

Removes duplicate elements from an RDD.
