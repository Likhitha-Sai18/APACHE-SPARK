# RDD Operations in Apache Spark – Structured Notes

---

# Introduction

Once an RDD is created, we can perform operations on it.

RDD operations are divided into two categories:

```text
           RDD Operations
                │
           ┌────┴────┐
           │         │
Transformations     Actions
```

---

# 1. Transformations

## Definition

Transformations are operations that create a **new RDD from an existing RDD**.

Since RDDs are immutable, transformations never modify the original RDD.

Instead:

```text
Old RDD
   ↓
Transformation
   ↓
New RDD
```

---

## Characteristics of Transformations

✔ Create a new RDD

✔ Do not modify the original RDD

✔ Lazy in nature

✔ Execution is delayed until an action is called

---

## Example

```python
rdd1 = sc.parallelize([1,2,3,4])

rdd2 = rdd1.map(lambda x: x*x)
```

### Result

```text
RDD1 = [1,2,3,4]

RDD2 = [1,4,9,16]
```

RDD1 remains unchanged.

---

# 2. Actions

## Definition

Actions are operations that trigger computation and return results.

Unlike transformations:

* No new RDD is created.
* Results are returned to the Driver Program.

---

## Characteristics of Actions

✔ Trigger execution

✔ Return final result

✔ Send output to Driver

✔ End the transformation chain

---

## Example

```python
rdd.collect()
```

### Output

```text
[1,4,9,16]
```

This is a Python list, not an RDD.

---

# Transformation vs Action

| Feature              | Transformation  | Action             |
| -------------------- | --------------- | ------------------ |
| Creates New RDD      | Yes             | No                 |
| Executes Immediately | No              | Yes                |
| Returns              | New RDD         | Final Result       |
| Lazy Evaluation      | Yes             | No                 |
| Example              | map(), filter() | collect(), count() |

---

# Why Don't Transformations Execute Immediately?

Because Spark uses **Lazy Evaluation**.

---

# Lazy Evaluation

## Definition

Lazy Evaluation is a strategy where Spark delays execution until an action is called.

---

## Why Lazy Evaluation?

Instead of executing every operation immediately:

```text
Read Data
Execute

Filter Data
Execute

Sort Data
Execute
```

Spark stores the plan:

```text
Read
 ↓
Filter
 ↓
Sort
```

and executes everything together when an action is called.

---

## Advantages

### Optimization

Spark optimizes the execution plan.

### Better Performance

Reduces unnecessary computations.

### Reduced Resource Usage

Less CPU and memory consumption.

---

# Example of Lazy Evaluation

```python
rdd1 = sc.parallelize([1,2,3,4])

rdd2 = rdd1.map(lambda x: x*x)

rdd3 = rdd2.filter(lambda x: x%2==0)
```

At this point:

❌ Nothing executes.

Spark only creates an execution plan.

---

## Action Trigger

```python
rdd3.collect()
```

Now Spark executes:

```text
RDD Creation
      ↓
Map
      ↓
Filter
      ↓
Collect
```

and returns the result.

---

# Example Flow

Suppose:

```text
Data = [1,2,3,4]
```

---

## Step 1: Create RDD

```python
rdd = sc.parallelize([1,2,3,4])
```

---

## Step 2: Square Numbers

```python
rdd1 = rdd.map(lambda x:x*x)
```

Result (Not Executed Yet):

```text
[1,4,9,16]
```

---

## Step 3: Filter Even Numbers

```python
rdd2 = rdd1.filter(lambda x:x%2==0)
```

Result (Still Not Executed):

```text
[4,16]
```

---

## Step 4: Execute Action

```python
rdd2.collect()
```

Output:

```text
[4,16]
```

Now all previous transformations are executed.

# Common Transformation Functions

---

# 1. map()

## Definition

Applies a function to every element of an RDD.

---

## Syntax

```python
rdd.map(function)
```

---

## Example

```python
rdd = sc.parallelize([1,2,3])

result = rdd.map(lambda x:x*2)
```

Output:

```text
[2,4,6]
```

---

## String Example

```python
rdd = sc.parallelize(["spark","hadoop"])

result = rdd.map(lambda x:x.upper())
```

Output:

```text
['SPARK','HADOOP']
```

---

# 2. flatMap()

## Definition

Similar to map() but flattens the output.

---

## Difference Between map() and flatMap()

### map()

```python
rdd = sc.parallelize(["Big Data"])

rdd.map(lambda x:x.split())
```

Output:

```text
[['Big','Data']]
```

List inside list.

---

### flatMap()

```python
rdd.flatMap(lambda x:x.split())
```

Output:

```text
['Big','Data']
```

Single flattened list.

---

## Interview Shortcut

```text
map()      → Nested Output

flatMap()  → Flattened Output
```

---

# 3. filter()

## Definition

Returns only elements satisfying a condition.

---

## Syntax

```python
rdd.filter(condition)
```

---

## Example

```python
rdd = sc.parallelize([1,2,3,4,5])

rdd.filter(lambda x:x%2==0)
```

Output:

```text
[2,4]
```

---

## String Example

```python
rdd = sc.parallelize([
"Big Data",
"Machine Learning",
"Data Science"
])

rdd.filter(lambda x:"Data" in x)
```

Output:

```text
['Big Data','Data Science']
```

---

# Common Action Functions

---

# 1. collect()

## Definition

Returns all elements of an RDD.

---

## Syntax

```python
rdd.collect()
```

---

## Example

```python
rdd = sc.parallelize([1,2,3])

rdd.collect()
```

Output:

```text
[1,2,3]
```

---

## Important

Avoid collect() on very large datasets.

Reason:

```text
Entire Dataset
       ↓
Driver Memory
```

May cause memory issues.

---

# 2. take()

## Definition

Returns the first N elements.

---

## Syntax

```python
rdd.take(n)
```

---

## Example

```python
rdd.take(2)
```

Output:

```text
[1,2]
```

---

## Advantage

More efficient than collect().

Only retrieves required data.

---

# 3. count()

## Definition

Returns the number of elements in the RDD.

---

## Syntax

```python
rdd.count()
```

---

## Example

```python
rdd = sc.parallelize([1,2,3])

rdd.count()
```

Output:

```text
3
```

---

# Summary of Common Transformations

| Transformation | Purpose                            |
| -------------- | ---------------------------------- |
| map()          | Apply function to every element    |
| flatMap()      | Apply function and flatten result  |
| filter()       | Select elements based on condition |

---

# Summary of Common Actions

| Action    | Purpose                 |
| --------- | ----------------------- |
| collect() | Return all elements     |
| take(n)   | Return first N elements |
| count()   | Count elements          |

---

# Complete Execution Flow

```text
Create RDD
      ↓
Transformation
(map)
      ↓
Transformation
(filter)
      ↓
Transformation
(flatMap)
      ↓
Action
(collect)
      ↓
Execution Starts
      ↓
Result Returned
```

# Interview Questions

### Q1. What are RDD operations?

RDD operations are divided into:

* Transformations
* Actions

---

### Q2. What is a Transformation?

An operation that creates a new RDD from an existing RDD.

---

### Q3. What is an Action?

An operation that triggers execution and returns results.

---

### Q4. Why are transformations called lazy?

Because they do not execute immediately and wait for an action.

---

### Q5. What is Lazy Evaluation?

A Spark optimization technique where execution is delayed until an action is called.

---

### Q6. Difference between map() and flatMap()?

| map()                    | flatMap()                   |
| ------------------------ | --------------------------- |
| Returns nested structure | Returns flattened structure |

---

### Q7. Which action returns all elements?

```python
collect()
```

---

### Q8. Which action is more efficient than collect() for viewing data?

```python
take()
```

## Memory Trick

### Transformations = MFF

**M** → map()
**F** → flatMap()
**F** → filter()

### Actions = CTC

**C** → collect()
**T** → take()
**C** → count()

👉 **Transformations create RDDs, Actions produce Results.** This is one of the most frequently asked Spark interview concepts. 🚀
