---

# Introduction to RDD

**RDD (Resilient Distributed Dataset)** is the fundamental data structure of Apache Spark.

It is Spark's primary abstraction for storing and processing data in a distributed manner.

### Definition

> RDD is an immutable, distributed collection of objects that can be processed in parallel across a cluster.

---

# Why RDD?

Before Spark, Hadoop MapReduce processed data directly from disk.

Spark introduced RDDs to:

* Enable distributed processing
* Support fault tolerance
* Improve performance through parallel execution

---

# Simple Analogy

If you are familiar with Pandas:

```text id="4qagjc"
Pandas      → DataFrame
Spark       → RDD (basic abstraction)
```

Think of RDD as a distributed collection of data spread across multiple machines.

---

# Characteristics of RDD

RDD has three important properties:

```text id="kk7l8g"
RDD

R → Resilient
D → Distributed
D → Dataset
```

---

# 1. Distributed

## Meaning

RDD data is distributed across multiple machines (nodes) in a cluster.

Instead of storing all data on one machine:

```text id="is2iyk"
Machine 1
 100 GB Data
```

Spark distributes it as:

```text id="yjlwm9"
Node 1 → 25 GB
Node 2 → 25 GB
Node 3 → 25 GB
Node 4 → 25 GB
```

---

## Advantage

Multiple machines process data simultaneously.

```text id="8g1zwu"
Node 1 → Task 1
Node 2 → Task 2
Node 3 → Task 3
Node 4 → Task 4
```

This is called **Parallel Processing**.

---

# 2. Immutable

## Meaning

Once an RDD is created, it cannot be modified.

❌ Not Allowed:

```text id="2hbhaw"
Modify Existing RDD
```

✔ Allowed:

```text id="v44oxk"
Create New RDD
from Existing RDD
```

---

## Example

```python
rdd1 = sc.parallelize([1,2,3,4])

rdd2 = rdd1.map(lambda x: x*2)
```

### Result

```text
RDD1 = [1,2,3,4]

RDD2 = [2,4,6,8]
```

RDD1 remains unchanged.

---

## Advantage

Immutability provides:

* Fault Tolerance
* Consistency
* Easier Parallel Processing

---

# 3. Resilient

## Meaning

RDD can recover automatically from failures.

Suppose:

```text id="feyv9v"
Node 1 ✓
Node 2 ✗ Failed
Node 3 ✓
```

Spark reconstructs lost data using lineage information.

---

## Advantage

* Fault Tolerant
* Reliable
* Suitable for large clusters

---

# RDD Partitioning

## What is a Partition?

A partition is a logical division of data within an RDD.

---

### Example

Suppose we have a file:

```text
1
2
3
4
5
6
7
8
```

Spark may divide it as:

```text id="zk4l7t"
Partition 1 → [1,2]

Partition 2 → [3,4]

Partition 3 → [5,6]

Partition 4 → [7,8]
```

---

## Why Partition?

Partitions allow:

* Parallel Processing
* Better Resource Utilization
* Faster Computation

---

## Processing Example

```text id="6uvxhm"
Partition 1 → Executor 1

Partition 2 → Executor 2

Partition 3 → Executor 3

Partition 4 → Executor 4
```

All executors process simultaneously.

---

# Creating RDDs

Spark provides two major methods to create RDDs.

---

# Method 1: Creating RDD from Collection

## Definition

Create an RDD from a Python collection.

Examples:

* List
* Tuple
* Set

---

## Syntax

```python
rdd = sc.parallelize(collection)
```

---

## Example

```python
from pyspark import SparkContext

sc = SparkContext()

data = [10,20,30,40]

rdd = sc.parallelize(data)

print(rdd.collect())
```

---

### Output

```text
[10,20,30,40]
```

---

# Specifying Number of Partitions

## Syntax

```python
rdd = sc.parallelize(data, numSlices)
```

---

## Example

```python
data = [1,2,3,4,5,6]

rdd = sc.parallelize(data, 3)
```

---

### Result

```text id="h0zv4p"
Partition 1 → [1,2]

Partition 2 → [3,4]

Partition 3 → [5,6]
```

---

# Method 2: Creating RDD from External Sources

## Definition

RDDs can also be created by reading data from external storage systems.

---

## Supported Sources

* Local Files
* HDFS
* S3
* Hive
* Databases
* CSV Files
* Text Files

---

## Syntax

```python
rdd = sc.textFile(path)
```

---

## Example

```python
from pyspark import SparkContext

sc = SparkContext()

rdd = sc.textFile("employees.txt")
```

---

## Minimum Partitions

We can specify the minimum number of partitions.

### Syntax

```python
rdd = sc.textFile(path, minPartitions)
```

---

### Example

```python
rdd = sc.textFile(
      "employees.txt",
      minPartitions=5
)
```

Spark creates at least 5 partitions.

---

# Parallel Processing with RDD

Suppose file contains:

```text
100 Million Records
```

RDD splits it:

```text id="jlwmwd"
RDD
 │
 ├── Partition 1
 ├── Partition 2
 ├── Partition 3
 └── Partition 4
```

Each partition is processed by different executors.

```text id="g9o7go"
Executor 1 → Partition 1

Executor 2 → Partition 2

Executor 3 → Partition 3

Executor 4 → Partition 4
```

Result:

✔ Faster Processing

✔ Better Scalability

# Advantages of RDD

### Fault Tolerance

Recovers automatically using lineage.

### Distributed Processing

Data spread across multiple nodes.

### Parallel Execution

Partitions processed simultaneously.

### Immutable Nature

Prevents accidental modification.

### Scalability

Can handle huge datasets.

---

# Limitations of RDD

### No Query Optimization

Slower than DataFrames and Datasets.

### More Memory Usage

Stores additional metadata.

### Complex Syntax

Requires more code compared to DataFrames.

---

# RDD vs DataFrame

| Feature           | RDD      | DataFrame |
| ----------------- | -------- | --------- |
| Abstraction Level | Low      | High      |
| Optimization      | No       | Yes       |
| Performance       | Slower   | Faster    |
| Schema            | No       | Yes       |
| Ease of Use       | Moderate | Easy      |
| Recommended Today | Rarely   | Yes       |
