# Apache Spark Caching and Persistence – Structured Notes

---

# Introduction

In Apache Spark, transformations are evaluated using **Lazy Evaluation**.

This means:

* Transformations are not executed immediately.
* Execution happens only when an **Action** is called.
* Every action may trigger recomputation of the entire transformation lineage.

For large datasets, repeatedly performing the same computations can waste time and resources.

To overcome this problem, Spark provides:

1. **Caching**
2. **Persistence**

These techniques improve Spark performance by storing intermediate results and reusing them whenever required.

---

# Why Do We Need Caching and Persistence?

Consider the following Spark workflow:

```text
Read Data
    ↓
Tokenize Data
    ↓
Create Pair RDD
    ↓
Perform Multiple Actions
```

Suppose we have:

```python
rdd = sc.textFile("data.txt")

words = rdd.flatMap(lambda x: x.split())

pairRDD = words.map(lambda x:(x,1))
```

Now perform Action 1:

```python
pairRDD.count()
```

Spark executes:

```text
Read Data
    ↓
Tokenize
    ↓
Create PairRDD
    ↓
Count
```

---

Now perform Action 2:

```python
pairRDD.collect()
```

Spark again executes:

```text
Read Data
    ↓
Tokenize
    ↓
Create PairRDD
    ↓
Collect
```

---

Now perform Action 3:

```python
pairRDD.take(5)
```

Spark once again executes:

```text
Read Data
    ↓
Tokenize
    ↓
Create PairRDD
    ↓
Take
```

---

# Problem

The same computations are executed repeatedly.

For Big Data workloads this leads to:

* Increased execution time
* Unnecessary CPU usage
* Additional memory consumption
* Resource wastage

---

# Solution

Store the intermediate result once and reuse it.

This is achieved using:

```text
Cache
Persist
```

---

# What is Caching?

## Definition

Caching is a Spark optimization technique that stores an RDD in memory after its first computation.

Instead of recomputing the entire lineage every time, Spark reuses the cached data.

---

# Syntax

```python
rdd.cache()
```

---

# Example

```python
rdd = sc.textFile("data.txt")

words = rdd.flatMap(
    lambda x:x.split()
)

pairRDD = words.map(
    lambda x:(x,1)
)

pairRDD.cache()
```

---

# Working of Cache

### First Action

```python
pairRDD.count()
```

Spark computes the RDD and stores it in memory.

```text
Read Data
    ↓
Transformations
    ↓
PairRDD
    ↓
Store in Memory
```

---

### Second Action

```python
pairRDD.collect()
```

Spark directly uses the cached result.

```text
Memory
   ↓
Collect
```

No recomputation occurs.

---

### Third Action

```python
pairRDD.take(5)
```

Again Spark uses cached data.

```text
Memory
   ↓
Take
```

---

# Benefits of Cache

### Faster Execution

Data is reused from memory.

### Avoids Recomputation

Transformations are executed only once.

### Saves Resources

Less CPU and disk activity.

### Improves Performance

Particularly useful when the same RDD is used multiple times.

---

# What is Persistence?

## Definition

Persistence is a Spark optimization technique that stores intermediate RDD results so they can be reused later instead of being recomputed.

Like caching, it avoids repeated execution of transformations.

---

# Syntax

```python
rdd.persist()
```

---

# Example

```python
pairRDD.persist()
```

---

# Working of Persist

### First Action

```python
pairRDD.count()
```

Spark computes and stores the RDD.

---

### Subsequent Actions

```python
pairRDD.collect()

pairRDD.take(5)
```

Spark reuses the stored result.

No need to execute all previous transformations again.

---

# Cache vs Persist

| Feature              | Cache                      | Persist                    |
| -------------------- | -------------------------- | -------------------------- |
| Purpose              | Store intermediate results | Store intermediate results |
| Avoids Recomputation | Yes                        | Yes                        |
| Improves Performance | Yes                        | Yes                        |
| Reuses Computed Data | Yes                        | Yes                        |
| Syntax               | `cache()`                  | `persist()`                |

---

# Important Point

Both caching and persistence are used to:

```text
Store Intermediate Results
            ↓
Avoid Repeated Computation
            ↓
Improve Spark Performance
```

---

# When Should We Use Cache/Persist?

Use them when:

✅ Same RDD is used multiple times

✅ Multiple actions are performed on the same dataset

✅ Expensive computations need to be reused

---

Avoid using them when:

❌ RDD is used only once

❌ Dataset is very small

---

# Real-World Example

Suppose a company has employee data.

```text
Read Employee Data
        ↓
Create PairRDD
        ↓
Find Total Employees
Find Employees by Department
Find Employees by City
Find Employees by Salary Range
```

Without caching:

Each query starts from reading data again.

With caching:

Data is loaded and processed once, then reused for all queries.

Result:

```text
Faster Queries
Lower Resource Usage
Better Performance
```
