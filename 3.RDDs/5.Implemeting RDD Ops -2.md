# 1. flatMap() Transformation

## Definition

flatMap() is similar to map(), but it performs a **one-to-many transformation**.

### Key Difference

| map()                  | flatMap()                     |
| ---------------------- | ----------------------------- |
| One Input → One Output | One Input → Multiple Outputs  |
| Returns List of Lists  | Returns Single Flattened List |

---

# Why Do We Need flatMap()?

Suppose we have:

```text
John HR India
Sarah Finance Australia
```

Using map():

```python
rdd.map(lambda x: x.split())
```

Output:

```text
[
 ['John','HR','India'],
 ['Sarah','Finance','Australia']
]
```

Result is a list of lists.

---

Using flatMap():

```python
rdd.flatMap(lambda x: x.split())
```

Output:

```text
[
 'John',
 'HR',
 'India',
 'Sarah',
 'Finance',
 'Australia'
]
```

The nested structure is flattened.

---


Output:

```text

```

---

# Example

## Create RDD

```python
data = [
"Big Data",
"Apache Spark"
]

rdd = sc.parallelize(data)
```

---

## map()

```python
rdd.map(lambda x:x.split()).collect()
```

Output:

```text
[
 ['Big','Data'],
 ['Apache','Spark']
]
```

---

## flatMap()

```python
rdd.flatMap(lambda x:x.split()).collect()
```

Output:

```text
[
 'Big',
 'Data',
 'Apache',
 'Spark'
]
```

---

### Tokenization

Converting sentences into words.

---

### Text Processing

Most NLP applications use flatMap().

# map() vs flatMap()

| Feature             | map()             | flatMap()       |
| ------------------- | ----------------- | --------------- |
| Transformation Type | One-to-One        | One-to-Many     |
| Output              | Nested            | Flattened       |
| Common Use          | Record Processing | Word Processing |

# 2. union() Transformation

## Definition

union() combines two RDDs into a single RDD.

It behaves similarly to the UNION operation in Set Theory.

---

# Scenario

Analytics20 has:

```text
India Branch
Dubai Branch
```

We have:

```text
analyticsIndiaRDD
analyticsDubaiRDD
```

---

# Example

## India Locations

```text
India
Australia
Dubai
Scotland
```

---

## Dubai Locations

```text
India
Australia
Dubai
Scotland
South Africa
```

---

## Apply Union

```python
all_places = india_places.union(
               dubai_places
             )
```

---

## Output

```text
India
Australia
Dubai
Scotland
India
Australia
Dubai
Scotland
South Africa
```

Notice:

✔ Data from both RDDs included

✔ Duplicates remain

---

# Important Point

### union() Does NOT Remove Duplicates

Duplicates are preserved.

If unique values are needed:

```python
rdd1.union(rdd2).distinct()
```

# 3. intersection() Transformation

## Definition

intersection() returns only common elements present in both RDDs.

---

# Example

## India Locations

```text
India
Australia
Dubai
Scotland
```

---

## Dubai Locations

```text
India
Australia
Dubai
Scotland
South Africa
```

---

## Apply Intersection

```python
common_places = india_places.intersection(
                    dubai_places
                )
```

---

## Output

```text
India
Australia
Dubai
Scotland
```

---

## Visual Representation

```text
RDD 1
------
India
Australia
Dubai
Scotland

RDD 2
------
India
Australia
Dubai
South Africa

Intersection
------------
India
Australia
Dubai
```

Only common elements remain.

---

# Difference Between Union and Intersection

| Feature     | union()      | intersection()        |
| ----------- | ------------ | --------------------- |
| Purpose     | Combine RDDs | Find Common Elements  |
| Duplicates  | Allowed      | Removed Automatically |
| Output Size | Larger       | Smaller               |

# Complete Analytics20 Example

---

## Step 1: Read India Data

```python
india = sc.textFile(
    "analytics20india.txt"
)
```

---

## Step 2: Tokenize

```python
india_map = india.map(
    lambda x:x.split()
)
```

---

## Step 3: Extract Places

```python
india_places = india_map.map(
    lambda x:x[2]
)
```

---

## Step 4: Unique Places

```python
india_places_distinct = \
india_places.distinct()
```

---

## Step 5: Read Dubai Data

```python
dubai = sc.textFile(
    "analytics20dubai.txt"
)
```

---

## Step 6: Tokenize

```python
dubai_map = dubai.map(
    lambda x:x.split()
)
```

---

## Step 7: Extract Places

```python
dubai_places = dubai_map.map(
    lambda x:x[2]
)
```

---

## Step 8: Distinct Places

```python
dubai_places_distinct = \
dubai_places.distinct()
```

---

## Step 9: Union

```python
all_places = \
india_places_distinct.union(
dubai_places_distinct
)
```

Output:

```text
India
Australia
Dubai
Scotland
India
Australia
Dubai
Scotland
South Africa
```

---

## Step 10: Intersection

```python
common_places = \
india_places_distinct.intersection(
dubai_places_distinct
)
```

Output:

```text
India
Australia
Dubai
Scotland
```

# Transformation Summary

| Transformation | Purpose                    |
| -------------- | -------------------------- |
| map()          | One-to-One Transformation  |
| flatMap()      | One-to-Many Transformation |
| filter()       | Filter Records             |
| distinct()     | Remove Duplicates          |
| union()        | Combine RDDs               |
| intersection() | Find Common Elements       |

---

# Interview Questions

### Q1. What is the difference between map() and flatMap()?

| map()                | flatMap()                  |
| -------------------- | -------------------------- |
| One Output per Input | Multiple Outputs per Input |
| Nested Output        | Flattened Output           |

---

### Q2. Does union() remove duplicates?

No.

Duplicates remain.

Example:

```text
India
India
```

Both are retained.

---

### Q3. How can duplicates be removed after union?

```python
rdd1.union(rdd2).distinct()
```

---

### Q4. What does intersection() do?

Returns common elements present in both RDDs.

---

### Q5. Which transformation is heavily used in Word Count programs?

```python
flatMap()
```

# Memory Trick

### MFDUI

**M** → map()
**F** → flatMap()
**D** → distinct()
**U** → union()
**I** → intersection()

These are the most commonly asked RDD transformations in Spark interviews and form the foundation for advanced Spark operations such as joins, aggregations, and DataFrame transformations. 🚀
