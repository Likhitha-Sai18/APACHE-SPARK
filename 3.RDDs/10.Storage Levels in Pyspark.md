# Apache Spark Storage Levels, Serialization & Persistence – Structured Notes

> **Note:** Before understanding Spark Storage Levels, we must understand **Serialization** and **Deserialization**, because several storage levels use serialized objects to save memory.

---

# 1. Serialization and Deserialization

## What is Serialization?

Serialization is the process of converting an object into a sequence of bytes.

These bytes can then be:

* Stored in memory
* Stored on disk
* Sent over a network

### Visualization

```text
Java/Python Object
       ↓
 Serialization
       ↓
Byte Stream
```

### Benefits

✅ Reduces memory consumption

✅ Improves network transfer speed

✅ Easier to store and transmit data

---

## What is Deserialization?

Deserialization is the reverse process.

It converts the byte stream back into the original object.

### Visualization

```text
Byte Stream
      ↓
Deserialization
      ↓
Original Object
```

---

## Drawback

Deserialization requires extra CPU processing.

```text
Serialized Data
       ↓
Convert Back
       ↓
Additional Processing Time
```

Therefore:

* Serialized objects use less memory
* Deserialized objects are faster to access

---

# 2. Why Storage Levels Exist?

When an RDD is persisted, Spark must decide:

```text
Where should data be stored?
```

Possible options:

```text
Memory
Disk
Memory + Disk
Serialized Format
Replicated Copies
```

Different storage levels provide different trade-offs between:

* Memory usage
* CPU usage
* Fault tolerance
* Performance

---

# 3. MEMORY_ONLY

## Definition

Stores RDD as deserialized Java objects in memory.

### Visualization

```text
RDD
 ↓
RAM
```

---

## Characteristics

✅ Fastest access

✅ No serialization overhead

❌ Uses more memory

❌ If memory becomes full, partitions are recomputed

---

## Example

```python
rdd.persist(StorageLevel.MEMORY_ONLY)
```

---

# 4. MEMORY_AND_DISK

## Definition

Stores RDD in memory.

If memory becomes insufficient:

```text
Extra Partitions
        ↓
      Disk
```

---

### Visualization

```text
RDD
 ↓
Memory
 ↓
If Full
 ↓
Disk
```

---

## Characteristics

✅ Prevents recomputation

✅ Can handle larger datasets

❌ Disk access is slower than memory

---

## Example

```python
rdd.persist(StorageLevel.MEMORY_AND_DISK)
```

---

# 5. MEMORY_ONLY_SER

## Definition

Stores RDD as serialized objects in memory.

### Visualization

```text
RDD
 ↓
Serialization
 ↓
RAM
```

---

## Characteristics

✅ Uses less memory

✅ Can store larger datasets

❌ Additional CPU needed for deserialization

❌ Slightly slower access

---

## Example

```text
Available only in Java/Scala
```

---

# 6. MEMORY_AND_DISK_SER

## Definition

Stores serialized RDD in memory.

If memory becomes full:

```text
Serialized Data
       ↓
Disk
```

---

## Characteristics

✅ Better space efficiency

✅ Supports larger datasets

❌ Serialization overhead

❌ Disk access overhead

---

# 7. DISK_ONLY

## Definition

Stores all partitions only on disk.

### Visualization

```text
RDD
 ↓
Disk
```

---

## Characteristics

✅ Very low memory usage

❌ Slowest access

❌ Heavy disk I/O

---

## Example

```python
rdd.persist(StorageLevel.DISK_ONLY)
```

---

# 8. Replicated Storage Levels

Examples:

```text
MEMORY_ONLY_2
MEMORY_AND_DISK_2
```

---

## Definition

Stores every partition on two different nodes.

### Visualization

```text
Partition 1
   ↓
Node A

Copy
   ↓
Node B
```

---

## Benefits

✅ Faster recovery from failures

✅ Better fault tolerance

---

## Drawback

❌ Requires more storage

---

# 9. OFF_HEAP Storage

## Definition

Stores serialized data outside JVM heap memory.

Uses:

```text
Tachyon / Alluxio
```

(memory-centric distributed storage systems)

---

### Visualization

```text
RDD
 ↓
Serialized
 ↓
Off Heap Memory
```

---

## Benefits

✅ Reduces JVM garbage collection overhead

✅ Better memory management

✅ Fast access

---

# What is Tachyon?

Tachyon (now called Alluxio) is:

```text
Memory-Centric
Distributed Storage System
```

It allows:

* Fast sharing of data
* In-memory data access
* Faster Spark processing

| Storage Level       | Memory          | Disk | Serialized | Speed   |
| ------------------- | --------------- | ---- | ---------- | ------- |
| MEMORY_ONLY         | ✅               | ❌    | ❌          | Fastest |
| MEMORY_AND_DISK     | ✅               | ✅    | ❌          | Fast    |
| MEMORY_ONLY_SER     | ✅               | ❌    | ✅          | Medium  |
| MEMORY_AND_DISK_SER | ✅               | ✅    | ✅          | Medium  |

# Storage Levels in PySpark

PySpark supports:

```python
StorageLevel.MEMORY_ONLY

StorageLevel.MEMORY_AND_DISK

StorageLevel.DISK_ONLY

StorageLevel.MEMORY_ONLY_2

StorageLevel.MEMORY_AND_DISK_2

StorageLevel.OFF_HEAP
```

---

## Missing in PySpark

These are mainly Java/Scala-specific:

```text
MEMORY_ONLY_SER

MEMORY_AND_DISK_SER
```

Reason:

PySpark already uses Python object serialization through Pickle.

---

# What is Pickle?

Pickle is Python's serialization library.

It converts Python objects into byte streams.

### Example

```python
import pickle

data = [1,2,3]

serialized = pickle.dumps(data)

deserialized = pickle.loads(serialized)
```

---

# How to Choose a Storage Level?

## Rule 1

If data comfortably fits in memory:

```text
Use MEMORY_ONLY
```

Best performance.

---

## Rule 2

If data does not fit entirely in memory:

```text
Use MEMORY_AND_DISK
```

Most common choice.

---

## Rule 3

Use serialized storage only when memory is limited.

```text
MEMORY_ONLY_SER
```

---

## Rule 4

Avoid spilling to disk unless necessary.

Disk operations are slower.

---

## Rule 5

Use replicated levels when fast recovery is required.

Example:

```text
Web Applications
Real-Time Systems
```

---

# Features of Persistence

---

## 1. Cost Efficient

Spark computations can be expensive.

Persistence avoids recomputation.

```text
Compute Once
Reuse Many Times
```

---

## 2. Time Efficient

Stored results are reused.

Less execution time.

---

## 3. Better Performance

Reduces:

* CPU usage
* Memory usage
* Network communication

---

# Changing Persistence Levels

## Check Cache Status

```python
rdd.is_cached
```

Output:

```python
False
```

---

## Persist an RDD

```python
from pyspark import StorageLevel

rdd.persist(
    StorageLevel.MEMORY_ONLY
)
```

---

## Verify

```python
rdd.is_cached
```

Output:

```python
True
```

---

## Remove Persistence

```python
rdd.unpersist()
```

---

## Change Storage Level

### Step 1

Remove old persistence.

```python
rdd.unpersist()
```

### Step 2

Apply new persistence.

```python
rdd.persist(
    StorageLevel.MEMORY_AND_DISK
)
```

---

# Practical Example

```python
from pyspark import StorageLevel

numbers = list(range(1,11))

rdd = sc.parallelize(numbers)
```

---

Check:

```python
rdd.is_cached
```

Output:

```python
False
```

---

Persist:

```python
rdd.persist(
StorageLevel.MEMORY_ONLY
)
```

---

Verify:

```python
rdd.is_cached
```

Output:

```python
True
```

---

Change Level:

```python
rdd.unpersist()

rdd.persist(
StorageLevel.MEMORY_AND_DISK
)
```

RDD is now stored using a different storage strategy.
