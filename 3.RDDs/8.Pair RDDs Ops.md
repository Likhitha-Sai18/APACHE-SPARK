# Dataset Description

Assume we have a file:

```text
students-data.txt
```

Each row contains:

```text
RollNo Section Name City Subject1 Subject2 Subject3 Subject4 Subject5
```

Example:

```text
101 A Ravi Delhi 80 75 90 85 88
102 B Kiran Mumbai 70 68 72 74 76
103 A Sneha Delhi 90 92 88 91 89
104 C Rahul Chennai 65 70 75 80 85
105 B Buncho Mumbai 78 82 80 76 79
```

---

# Step 1: Create Spark Context

```python
from pyspark import SparkContext

sc = SparkContext.getOrCreate()
```

---

# Step 2: Load File into RDD

```python
student_marks = sc.textFile("students-data.txt")
```

---

# View Data

```python

```

Output:

```python

'101 A Ravi Delhi 80 75 90 85 88',
'102 B Kiran Mumbai 70 68 72 74 76',
'103 A Sneha Delhi 90 92 88 91 89'
```

# Step 3: Split Records

Convert each line into a list.

```python
student_marks = student_marks.map(
    lambda x: x.split(" ")
```



# Creating Pair RDD

We will use:

```text
Key   = Roll Number
Value = (Section, Name, City)
```

---

```python
)
```

# Question: Count Students Per Section

Suppose we have:

```text
101 A Ravi
102 B Kiran
103 A Sneha
104 C Rahul
105 B Buncho
```

---

# Create Section Pair RDD

Key = Section

Value = 1

```python
section_rdd = student_marks.map(
    lambda x:(x[1],1)
)
```

Output:

```python
[
('A',1),
('B',1),
('A',1),
('C',1),
('B',1)
]
```

---

# Method 1: Using groupByKey()

```python
section_rdd.groupByKey() \
           .mapValues(len) \
           .collect()
```

Output:

```python
[
('A',2),
('B',2),
('C',1)
]
```

---

# How groupByKey Works

### Step 1

Input

```text
(A,1)
(B,1)
(A,1)
(C,1)
(B,1)
```

---

### Step 2

Group Values

```text
A → [1,1]

B → [1,1]

C → [1]
```

---

### Step 3

Count Values

```text
A → 2

B → 2

C → 1
```

---

# Method 2: Using reduceByKey()

More efficient.

```python
section_rdd.reduceByKey(
    lambda x,y:x+y
).collect()
```

Output:

```python
[
('A',2),
('B',2),
('C',1)
]
```

---

# Why reduceByKey is Better

### groupByKey

```text
Move ALL data
      ↓
Group
      ↓
Count
```

Large network shuffle.

---

### reduceByKey

```text
Aggregate Locally
      ↓
Move Less Data
      ↓
Final Aggregation
```

Much faster.

---

# Performance Comparison

## groupByKey()

```python
section_rdd.groupByKey() \
           .mapValues(len)
```

### Characteristics

❌ High shuffle

❌ More memory

❌ Slower

---

## reduceByKey()

```python
section_rdd.reduceByKey(
lambda x,y:x+y
)
```

### Characteristics

✅ Less shuffle

✅ Less memory

✅ Faster

---

# Interview Question

### Which is preferred?

```text
reduceByKey()
```

because it reduces data before shuffling.

# Summary Table

| Operation     | Type           | Purpose             |
| ------------- | -------------- | ------------------- |
| keys()        | Transformation | Extract Keys        |
| values()      | Transformation | Extract Values      |
| lookup()      | Action         | Find values for key |
| groupByKey()  | Transformation | Group values        |
| reduceByKey() | Transformation | Aggregate values    |
| collect()     | Action         | Return result       |

# Important Interview Point

When asked:

> "Count students in each section"

Always prefer:

```python
student_marks.map(
lambda x:(x[1],1)
).reduceByKey(
lambda x,y:x+y
)
```

instead of `groupByKey()` because it is more scalable and efficient for Big Data workloads. 🚀

Make the aspect ratio 16:9
