# Spark Execution Internals: Jobs, Stages, Tasks & Partitions

This module is one of the most important concepts in Spark because it explains **what happens behind the scenes when you run a Spark application**.

Understanding Spark execution helps you:

✅ Write optimized Spark code

✅ Reduce execution time

✅ Use cluster resources efficiently

✅ Troubleshoot performance issues

✅ Crack Spark interviews

---

# Big Picture of Spark Execution

When you write Spark code:

```python
rdd = sc.textFile("data.txt")

result = (
    rdd
    .map(lambda x: x.split(","))
    .filter(lambda x: int(x[2]) > 100)
    .map(lambda x: (x[0], int(x[2])))
    .reduceByKey(lambda a,b: a+b)
)

result.collect()
```

Spark does not immediately execute every transformation.

Instead Spark creates an execution plan and executes it only when an action is called.

---

## Spark Execution Hierarchy

Think of Spark execution as:

```text
Application
    ↓
Job
    ↓
Stage
    ↓
Task
```

---

# 1. Spark Application

A Spark Application is the complete program written by the user.

Example:

```python
rdd = sc.textFile("file.txt")

rdd.map(...)

rdd.filter(...)

rdd.collect()
```

Everything together forms a single Spark Application.

---

# 2. Spark Job

A Job is created whenever an Action is executed.

Examples of Actions:

```python
collect()
count()
take()
saveAsTextFile()
show()
```

---

### Example

```python
rdd = sc.textFile("file.txt")

rdd2 = rdd.map(lambda x:x.upper())

rdd3 = rdd2.filter(lambda x:"A" in x)

rdd3.collect()
```

When:

```python
collect()
```

is executed

Spark creates:

```text
1 Job
```

---

## Multiple Actions = Multiple Jobs

```python
rdd.count()

rdd.collect()
```

Spark creates:

```text
Job 1 → count()

Job 2 → collect()
```

because each action triggers execution.

---

# Visual Representation

```text
Spark Application
         |
         V
      Job 1
```

---

# 3. Spark Stage

A Job is divided into one or more Stages.

Spark creates new stages whenever:

```text
Data Shuffle occurs
```

---

## What is Shuffle?

Shuffle means:

```text
Moving data across machines
through the network
```

This is one of the most expensive Spark operations.

---

## Transformations Causing Shuffle

Examples:

```python
reduceByKey()
groupByKey()
join()
distinct()
repartition()
sortByKey()
```

These require data movement.

---

# Example Without Shuffle

```python
rdd = sc.textFile("data.txt")

rdd2 = rdd.map(...)

rdd3 = rdd.filter(...)

rdd3.collect()
```

Execution:

```text
Job
 |
 |-- Stage 1
```

Only one stage because no shuffle exists.

---

# Example With Shuffle

```python
rdd = sc.textFile("data.txt")

rdd2 = rdd.map(...)

rdd3 = rdd.filter(...)

rdd4 = rdd.reduceByKey(...)
```

Execution:

```text
Job
 |
 +-- Stage 1
 |
 +-- Stage 2
```

Why?

Because:

```python
reduceByKey()
```

causes shuffle.

---

# Stage Boundary

```text
RDD1
  |
 map()
  |
RDD2
  |
 filter()
  |
RDD3
  |
 reduceByKey()
  | ← Shuffle Here
RDD4
```

Spark creates:

```text
Stage 1:
RDD1 → RDD2 → RDD3

Stage 2:
RDD4
```

---

# Pipeline Execution

Inside a stage:

```python
map()
filter()
flatMap()
```

are combined together.

This is called:

# Pipelining

Instead of:

```text
Map → store result
Filter → store result
FlatMap → store result
```

Spark performs:

```text
Map + Filter + FlatMap
in one pass
```

This reduces:

* Memory usage
* Disk I/O
* CPU overhead

and improves performance.

---

# Narrow vs Wide Transformation

Understanding this concept explains stage creation.

---

## Narrow Transformation

Each child partition depends on only one parent partition.

Example:

```python
map()
filter()
flatMap()
```

Visualization:

```text
Partition 1 → Partition 1

Partition 2 → Partition 2
```

No network transfer.

---

## Wide Transformation

Child partition depends on multiple parent partitions.

Example:

```python
reduceByKey()
join()
groupByKey()
```

Visualization:

```text
P1 \
     \
      → New Partition
     /
P2 /
```

Requires shuffle.

Creates new stage.

---

# 4. Spark Task

A Stage is divided into multiple Tasks.

---

## What is a Task?

A Task is:

```text
Computation performed
on one partition
```

---

# Important Rule

```text
1 Partition = 1 Task
```

---

# Example

Suppose an RDD contains:

```text
4 Partitions
```

```text
P1
P2
P3
P4
```

Then Spark creates:

```text
Task 1
Task 2
Task 3
Task 4
```

One task per partition.

---

# Visual Representation

```text
Stage
 |
 +-- Task 1 (Partition 1)
 |
 +-- Task 2 (Partition 2)
 |
 +-- Task 3 (Partition 3)
 |
 +-- Task 4 (Partition 4)
```

---

# Task Execution on CPU Cores

Each task runs on one CPU core.

Example:

Cluster:

```text
Worker Node

Core 1
Core 2
Core 3
Core 4
```

RDD:

```text
4 Partitions
```

Execution:

```text
Task 1 → Core 1

Task 2 → Core 2

Task 3 → Core 3

Task 4 → Core 4
```

All execute simultaneously.

---

# Why More Partitions Increase Parallelism

Suppose:

```text
8 Cores
```

and

```text
8 Partitions
```

Then:

```text
8 Tasks run in parallel
```

Maximum CPU utilization.

---

# Interview Questions

### Q1. What triggers a Spark Job?

**Answer:**

An Action triggers a Spark Job.

Examples:

```python
collect()
count()
show()
```

---

### Q2. What triggers a new Stage?

**Answer:**

A Shuffle operation triggers a new Stage.

Examples:

```python
reduceByKey()
join()
groupByKey()
```

---

### Q3. What triggers a Task?

**Answer:**

Each partition generates one task.

```text
1 Partition = 1 Task
```

---

### Q4. Difference between Job and Stage?

| Job                 | Stage                |
| ------------------- | -------------------- |
| Triggered by Action | Triggered by Shuffle |
| Consists of stages  | Consists of tasks    |

---

### Q5. Difference between Stage and Task?

| Stage               | Task                    |
| ------------------- | ----------------------- |
| Collection of tasks | Smallest execution unit |
| Runs on cluster     | Runs on single core     |

### Remember

✅ Action → Job

✅ Shuffle → New Stage

✅ Partition → Task

✅ Task runs on one CPU Core

✅ More Partitions = More Parallelism

✅ Narrow Transformation → Same Stage

✅ Wide Transformation → New Stage

✅ Pipeline Execution improves performance

This Jobs → Stages → Tasks → Partitions relationship is the foundation of Spark performance tuning and is one of the most frequently asked topics in PySpark and Data Engineering interviews.
