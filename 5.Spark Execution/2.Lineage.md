# Apache Spark Lineage (RDD Lineage) – Complete Conceptual Notes

This module introduces one of Spark's most powerful features:

# Spark Lineage

Lineage is the reason Spark is fault tolerant without needing excessive data replication.

---

# Why Do We Need Lineage?

Imagine you have a Spark application:

```python
rdd1 = sc.textFile("data.txt")

rdd2 = rdd1.map(...)

rdd3 = rdd2.filter(...)

rdd4 = rdd3.reduceByKey(...)
```

Now suppose:

```text
RDD3 is lost
```

because:

* Worker node crashes
* Executor fails
* Memory is lost
* Machine shuts down

Question:

❓ How will Spark recover RDD3?

---

# Traditional Systems Approach

Older distributed systems often:

```text
Store multiple copies
of data
```

Advantages:

✅ Easy recovery

Disadvantages:

❌ More storage

❌ More network traffic

❌ Expensive

---

# Spark's Approach

Spark uses:

```text
RDD Immutability
+
Lineage
```

to recover data.

---

# What is Lineage?

## Definition

Lineage is:

```text
The logical execution plan
that records how an RDD
was created from parent RDDs.
```

or

```text
A history of transformations
applied on RDDs.
```

---

## Simple Example

```python
rdd1 = sc.parallelize([1,2,3,4])

rdd2 = rdd1.map(lambda x:x*2)

rdd3 = rdd2.filter(lambda x:x>4)
```

Spark records:

```text
RDD1
 |
map()
 |
RDD2
 |
filter()
 |
RDD3
```

This graph is called:

```text
RDD Lineage Graph
```

---

# Key Point

Lineage stores:

✅ Transformations

✅ Parent-child relationships

✅ Dependency information

It does NOT store:

❌ Actual data

❌ Computed results

---

# Think of It Like a Recipe

Suppose you bake a cake.

Instead of saving:

```text
100 copies of cake
```

you save:

```text
Recipe
```

If the cake is lost:

```text
Follow recipe again
```

Spark does the same thing.

Instead of saving multiple copies of RDDs:

```text
Stores instructions
to recreate them
```

---

# Lineage Example

Consider:

```python
rdd1 = sc.parallelize([1,2,3])

rdd2 = rdd1.map(lambda x:x*2)

rdd3 = rdd2.filter(lambda x:x>2)

rdd4 = rdd3.map(lambda x:x+1)
```

Lineage:

```text
RDD1
 |
map
 |
RDD2
 |
filter
 |
RDD3
 |
map
 |
RDD4
```

Spark remembers the entire chain.

---

# How Lineage Provides Fault Tolerance

Suppose:

```text
RDD4 lost
```

Spark simply recomputes:

```text
RDD1
 ↓
RDD2
 ↓
RDD3
 ↓
RDD4
```

using the lineage graph.

---

# Why This Works?

Because RDDs are Immutable.

---

## What is Immutability?

Once created:

```text
RDD cannot be modified
```

Example:

```python
rdd1 = sc.parallelize([1,2,3])

rdd2 = rdd1.map(lambda x:x*2)
```

Spark creates:

```text
New RDD
```

It never changes:

```text
RDD1
```

---

# Therefore

Spark knows:

```text
RDD2 came from RDD1
using map()
```

This relationship never changes.

So recovery becomes easy.

---

# Lineage is Logical, Not Physical

This is an extremely important interview point.

---

## Logical Plan

Lineage only records:

```text
What transformations
were applied
```

Example:

```text
RDD1
 |
map()
 |
RDD2
 |
filter()
 |
RDD3
```

This is a logical description.

---

## Physical Plan

Physical execution contains:

```text
Jobs
Stages
Tasks
Executors
Partitions
```

Lineage DOES NOT contain these.

---

# Lineage vs Physical Execution

| Lineage                | Physical Execution   |
| ---------------------- | -------------------- |
| Logical Plan           | Physical Plan        |
| Transformation History | Actual Execution     |
| Parent Dependencies    | Jobs, Stages, Tasks  |
| Fault Recovery         | Resource Utilization |
| No Execution           | Actual Execution     |

---

# Important Clarification

Lineage:

```text
DOES NOT
```

create:

```text
Jobs
Stages
Tasks
```

It simply records:

```text
Transformation history
```

---

# Example

```python
rdd1 = sc.parallelize(data)

rdd2 = rdd1.map(...)

rdd3 = rdd2.filter(...)
```

Spark only records:

```text
RDD1
 ↓
RDD2
 ↓
RDD3
```

No stages yet.

No tasks yet.

No executors yet.

---

# When Are Jobs, Stages, Tasks Created?

Only when an Action is called.

Example:

```python
rdd3.collect()
```

Now Spark converts lineage into:

```text
Physical Execution Plan
```

which contains:

```text
Job
 ↓
Stage
 ↓
Task
```

---

# Viewing Lineage in Spark

Spark provides:

```python
toDebugString()
```

to visualize lineage.

---

# Example

```python
rdd1 = sc.parallelize([1,2,3])

rdd2 = rdd1.map(lambda x:x*2)

print(rdd2.toDebugString())
```

Output:

```text
PythonRDD
 | ParallelCollectionRDD
```

This shows:

```text
RDD2 depends on RDD1
```

# Dependency Tracking

Lineage tracks dependencies.

---

## Narrow Dependency

Example:

```python
map()
filter()
```

Visualization:

```text
P1 → P1

P2 → P2
```

Easy recovery.

---

## Wide Dependency

Example:

```python
reduceByKey()
groupByKey()
join()
```

Visualization:

```text
P1 \
     \
      New Partition
     /
P2 /
```

Requires shuffle.

More expensive recovery.

# Advantages of Lineage

### 1. Fault Tolerance

Lost RDDs can be recomputed.

---

### 2. Less Storage

No need to replicate every dataset.

---

### 3. Simpler Architecture

No complex recovery mechanism.

---

### 4. Efficient

Stores only transformation metadata.

---

# Limitations of Long Lineage

Suppose:

```text
1000 transformations
```

Lineage becomes huge.

Recovery becomes slower.

---

## Solution

Use:

```python
cache()
persist()
checkpoint()
```

Checkpoint stores data permanently and breaks long lineage chains.

# Interview Questions

### Q1. What is RDD Lineage?

**Answer:**

RDD Lineage is the logical execution plan that records the sequence of transformations used to create an RDD.

---

### Q2. Why is lineage important?

**Answer:**

It enables fault tolerance by allowing Spark to recompute lost RDDs.

---

### Q3. Does lineage store actual data?

**Answer:**

No.

It only stores transformation history and dependencies.

---

### Q4. Is lineage logical or physical?

**Answer:**

Logical execution plan.

---

### Q5. Does lineage create stages?

**Answer:**

No.

Stages are created later during physical execution.

---

### Q6. Which function displays lineage?

```python
rdd.toDebugString()
```
