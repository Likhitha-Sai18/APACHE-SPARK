# Apache Spark DAG (Directed Acyclic Graph) & Complete Spark Execution Flow


# What is a DAG?

DAG stands for:

```text
Directed Acyclic Graph
```

Let's break this down:

### Directed

Every operation has a direction.

```text
RDD1 → RDD2 → RDD3
```

Data flows in one direction.

---

### Acyclic

No loops are allowed.

Valid:

```text
RDD1 → RDD2 → RDD3
```

Invalid:

```text
RDD1 → RDD2 → RDD3
 ↑             ↓
 └─────────────┘
```

Spark never creates cycles.

---

### Graph

A collection of nodes and connections.

Example:

```text
RDD1
 |
RDD2
 |
RDD3
```

---

# What is DAG in Spark?

## Definition

A DAG is:

```text
A physical execution plan
generated from lineage
that represents how Spark
will execute a job.
```

Think of it as:

```text
Lineage = Recipe

DAG = Cooking Plan

Tasks = Actual Cooking
```

---

# Lineage vs DAG

Many beginners confuse these.

| Lineage                 | DAG                      |
| ----------------------- | ------------------------ |
| Logical Plan            | Physical Execution Plan  |
| Records transformations | Organizes execution      |
| Fault tolerance         | Performance optimization |
| Tracks dependencies     | Creates stages/tasks     |
| No execution            | Used for execution       |

---

# Example

```python
rdd0 = sc.parallelize(
    ["Hello Spark Spark"]
)

rdd1 = rdd0.flatMap(
    lambda x:x.split()
)

rdd2 = rdd1.map(
    lambda x:(x,1)
)

rdd3 = rdd2.reduceByKey(
    lambda a,b:a+b
)

rdd3.collect()
```

---

# Step 1: Lineage Creation

Spark first records:

```text
RDD0
 |
flatMap
 |
RDD1
 |
map
 |
RDD2
 |
reduceByKey
 |
RDD3
```

This is only lineage.

Nothing executes yet.

---

# Step 2: Action Triggered

```python
rdd3.collect()
```

Now Spark says:

```text
Okay, time to execute.
```

---

# Step 3: DAG Creation

The DAG Scheduler examines lineage.

It identifies:

```text
Narrow Transformations

flatMap()
map()
```

and

```text
Wide Transformation

reduceByKey()
```

---

# Why reduceByKey Creates a New Stage?

Because:

```python
reduceByKey()
```

causes:

```text
Shuffle
```

Data must move across executors.

Whenever shuffle occurs:

```text
New Stage Created
```

---

# DAG Visualization

```text

 |

 |
RDD1
 |

 |
RDD2
 |
=========== Shuffle ===========
 |

 |

```

---

# Stage Formation

Spark divides DAG into:

```text
Stage 0
--------
parallelize
flatMap
map

Stage 1
--------
reduceByKey
```

---

# Why Are flatMap and map in the Same Stage?

Because they are:

```text
Narrow Transformations
```

No data movement.

Spark pipelines them.

---

# Pipelining

Instead of:

```text
Read Data
↓
Map
↓
Store

Read Again
↓
FlatMap
↓
Store
```

Spark performs:

```text
Read Data
↓
Map + FlatMap Together
```

This is called:

# Pipeline Execution

and improves performance significantly.

# Why DAG is Important?

Without DAG:

```text
Spark would execute
every transformation
individually.
```

This would be slow.

---

# DAG Optimization

DAG Scheduler:

### Combines Transformations

```python
map()
filter()
flatMap()
```

into:

```text
Single Pipeline
```

---

### Minimizes Shuffles

Shuffle is expensive because:

```text
Network Transfer
Disk I/O
Serialization
```

are involved.

---

### Creates Optimal Stages

Instead of:

```text
10 Stages
```

Spark may create:

```text
2 Stages
```

using DAG optimization.

---

# DAG Scheduler

Spark contains:

```text
DAG Scheduler
```

This is one of Spark's core components.

---

## Responsibilities

### 1. Analyze Lineage

```text
RDD Dependencies
```

---

### 2. Build DAG

```text
Execution Graph
```

---

### 3. Create Stages

Based on:

```text
Shuffle Boundaries
```

---

### 4. Pipeline Operations

Combine narrow transformations.

---

### 5. Generate Tasks

For each partition.

---

# Task Scheduler

After DAG Scheduler creates stages:

```text
Stage 0
Stage 1
```

they are passed to:

```text
Task Scheduler
```

---

## Responsibilities

### Create Tasks

Remember:

```text
1 Partition = 1 Task
```

---

### Assign Tasks

To:

```text
Executors
```

running on worker nodes.

---

# Cluster Manager

Task Scheduler communicates with:

```text
Cluster Manager
```

Examples:

* YARN
* Kubernetes
* Standalone
* Mesos

---

## Responsibilities

### Resource Allocation

Provides:

```text
CPU
Memory
Executors
```

for Spark.

# Interview Questions

### Q1. What is a DAG?

**Answer:**

A Directed Acyclic Graph representing Spark's execution plan.

---

### Q2. Who creates the DAG?

**Answer:**

The DAG Scheduler.

---

### Q3. What is the role of DAG Scheduler?

**Answer:**

It analyzes lineage, creates DAGs, pipelines transformations, and divides jobs into stages.

---

### Q4. What creates stages?

**Answer:**

Shuffle boundaries.

---

### Q5. What creates tasks?

**Answer:**

Partitions.

```text
1 Partition = 1 Task
```

---

### Q6. Difference Between DAG Scheduler and Task Scheduler?

| DAG Scheduler       | Task Scheduler    |
| ------------------- | ----------------- |
| Creates stages      | Creates tasks     |
| Builds DAG          | Assigns tasks     |
| Optimizes execution | Manages execution |

---

### Q7. Why is DAG important?

**Answer:**

It optimizes execution by minimizing shuffles and pipelining transformations.
