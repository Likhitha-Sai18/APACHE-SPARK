# Apache Spark Cluster Resource Managers – Structured Notes

## Introduction

To execute applications in a distributed environment, Apache Spark requires a **Cluster Resource Manager**.

A Cluster Resource Manager is responsible for:

* Allocating resources (CPU, Memory)
* Launching Executors
* Managing Worker Nodes
* Monitoring Application Execution

Without a resource manager, Spark cannot efficiently run applications across multiple machines in a cluster.

---

# Recap: How Spark Runs in a Cluster

```text
User Application
        │
        ▼
 Driver Program
        │
        ▼

        │
        ▼

        │
        ▼
 Worker Nodes
        │
        ▼

        │
        ▼

        │
        ▼

```

# What is a Cluster?

## Definition

A **Cluster** is a group of interconnected computers that work together as a single system.

### Example

Instead of processing 1 TB of data on one machine:

```text
Computer 1
Computer 2
Computer 3
Computer 4
```

All machines work together and share the workload.

---

# Spark Application in a Cluster

A Spark Application consists of:

### Driver Program

* Runs the user code
* Creates SparkContext
* Schedules jobs

### Executors

* Execute tasks
* Store data in memory
* Return results

### Cluster Manager

* Allocates resources
* Launches executors

---

# Ways to Run Spark Applications

Spark applications can run in two modes:

## 1. Local Mode

Application runs on a single machine.

```text
Laptop
   │
   ▼
Spark Application
```

### Characteristics

* No distributed processing
* Uses machine's CPU cores
* Easy setup
* Ideal for beginners

### Use Cases

* Learning Spark
* Development
* Testing
* Debugging

### Advantages

✔ Easy to configure

✔ No cluster required

✔ Quick testing

### Disadvantages

✖ Limited processing power

✖ Cannot handle huge datasets

---

## 2. Cluster Mode

Application runs across multiple machines.

```text
Driver
   │
Cluster Manager
   │
 ┌──────┬──────┬──────┐
 ▼      ▼      ▼
Worker Worker Worker
```

### Characteristics

* Distributed Processing
* Multiple machines
* Large-scale data handling

### Use Cases

* Production systems
* Big Data analytics
* Machine Learning pipelines
* Real-time processing

### Advantages

✔ High scalability

✔ Faster processing

✔ Fault tolerance

### Disadvantages

✖ Complex setup

✖ Requires cluster infrastructure

---

# Cluster Resource Managers Supported by Spark

Spark supports multiple resource managers.

## Types

1. Spark Standalone
2. Hadoop YARN
3. Apache Mesos
4. Kubernetes (Modern deployments)

---

# 1. Spark Standalone

## Definition

Spark Standalone is Spark's built-in cluster manager.

It comes pre-packaged with Apache Spark.

---

## Architecture

```text
Master Node
      │
 ┌────┴────┐
 │         │
Worker   Worker
```

---

## Features

* Simple setup
* Lightweight
* Included with Spark
* Easy to manage

---

## Advantages

✔ Easy installation

✔ Minimal configuration

✔ Good for learning

✔ Suitable for testing

---

## Disadvantages

✖ Limited scalability

✖ Less suitable for very large production environments

---

## Best Use Cases

* Learning Spark
* Development
* Testing
* Small projects

---

# 2. Hadoop YARN

## Full Form

**YARN = Yet Another Resource Negotiator**

---

## Definition

YARN is Hadoop's resource management framework.

Spark can run on top of YARN.

---

## Architecture

```text
Spark
   │
   ▼
YARN
   │
   ▼
Hadoop Cluster
```

---

## Features

### Resource Sharing

Multiple applications can share the same cluster.

Example:

```text
Cluster
 ├── Spark
 ├── Hive
 ├── MapReduce
 └── HBase
```

---

### Dynamic Resource Allocation

YARN allocates resources as required.

---

### Scalability

Can manage thousands of nodes.

---

## Advantages

✔ Highly scalable

✔ Production-ready

✔ Efficient resource utilization

✔ Supports multiple frameworks

---

## Disadvantages

✖ More complex setup

✖ Requires Hadoop ecosystem

---

## Best Use Cases

* Enterprise environments
* Large Hadoop clusters
* Production workloads

---

# 3. Apache Mesos

## Definition

Apache Mesos is a distributed cluster manager capable of managing resources across multiple frameworks.

---

## Architecture

```text
Mesos Master
      │
 ┌────┴────┐
 │         │
Agent    Agent
```

---

## Features

* Resource sharing
* Cluster management
* Multi-framework support

---

## Historical Importance

Apache Mesos was the **first cluster manager supported by Spark**.

---

## Advantages

✔ Flexible resource sharing

✔ Supports multiple applications

---

## Disadvantages

✖ Less commonly used today

✖ Lower adoption than YARN

✖ Limited community usage

---

## Best Use Cases

* Specialized cluster environments

---

# 4. Kubernetes

## Definition

Kubernetes is a container orchestration platform that can manage Spark applications inside containers.

---

## Architecture

```text
Kubernetes Cluster
        │
 ┌──────┴──────┐
 │             │
Spark Pods  Spark Pods
```

---

## Features

* Container-based deployment
* Auto-scaling
* Cloud-native architecture

---

## Advantages

✔ Cloud friendly

✔ Highly scalable

✔ Modern deployment model

✔ Works well with Docker

---

## Best Use Cases

* AWS
* Azure
* GCP
* Containerized applications

# How Spark Runs on a Resource Manager

## Step 1

User submits Spark application.

```text
spark-submit my_app.py
```

↓

## Step 2

Driver Program starts.

↓

## Step 3

SparkContext connects to Cluster Manager.

↓

## Step 4

Cluster Manager allocates resources.

↓

## Step 5

Executors are launched.

↓

## Step 6

Tasks sent to executors.

↓

## Step 7

Executors process data.

↓

## Step 8

Results returned to Driver.

↓

## Step 9

Application completes.

| Scalability            | Low                | High         |

# Interview Questions

### Q1. What is a Cluster Resource Manager?

A software component responsible for allocating resources and managing executors in a cluster.

---

### Q2. Which resource managers are supported by Spark?

* Spark Standalone
* Hadoop YARN
* Apache Mesos
* Kubernetes

---

### Q3. What is YARN?

Yet Another Resource Negotiator, Hadoop's resource management framework.

---

### Q4. Which cluster manager is included with Spark?

Spark Standalone.

---

### Q5. Which resource manager is most commonly used in production?

Hadoop YARN (traditional Hadoop environments) and Kubernetes (modern cloud environments).

---

### Q6. Which was the first cluster manager supported by Spark?

Apache Mesos.

---

### Q7. Why is Local Mode used?

For learning, testing, and development.

---

### Q8. Why is Cluster Mode used?

For large-scale production applications and distributed processing.
