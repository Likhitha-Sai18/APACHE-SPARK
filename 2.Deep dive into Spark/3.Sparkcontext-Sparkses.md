# SparkContext and SparkSession – Structured Notes

---

# Introduction

To interact with Apache Spark, we need an **entry point**.

Spark provides two important entry points:

1. **SparkContext** (Spark 1.x and earlier)
2. **SparkSession** (Spark 2.x and later)

Understanding the difference between them is a very common interview topic.

---

# 1. SparkContext

## Definition

**SparkContext** is the primary entry point to Spark functionality.

It establishes the connection between:

```text
Spark Application
        ↕
   Spark Cluster
```

Whenever a Spark application starts, a **SparkContext** object is created.

---

## Why Do We Need SparkContext?

SparkContext tells Spark:

* Which cluster to connect to
* How much memory to use
* Application name
* Configuration settings

Without SparkContext, Spark cannot communicate with the cluster.

---

# Role of SparkContext

SparkContext is responsible for:

### 1. Job Submission

Submits jobs to the cluster.

```text
User Code
    ↓
SparkContext
    ↓
Cluster
```

---

### 2. Resource Allocation

Requests resources from the Cluster Manager.

```text
SparkContext
      ↓
Cluster Manager
      ↓
Executors
```

---

### 3. Task Scheduling

Divides jobs into:

```text
Job
 ↓
Stages
 ↓
Tasks
```

and schedules them.

---

### 4. Task Distribution

Sends tasks to executors.

---

### 5. Result Collection

Collects results from executors and returns them to the application.

---

# SparkContext Lifecycle

```text
Create SparkContext
         ↓
Connect to Cluster
         ↓
Submit Jobs
         ↓
Execute Tasks
         ↓
Collect Results
         ↓
Stop SparkContext
```

---

# Creating SparkContext

Before creating SparkContext, we need:

## SparkConf

### Definition

SparkConf stores configuration settings for Spark.

Examples:

* Application Name
* Master URL
* Memory Settings

---

## Example

### Step 1: Create SparkConf

```python
from pyspark import SparkConf

conf = SparkConf() \
        .setAppName("MyApp") \
        .setMaster("local")
```

---

### Step 2: Create SparkContext

```python
from pyspark import SparkContext

sc = SparkContext(conf=conf)
```

---

# Complete Example

```python
from pyspark import SparkConf
from pyspark import SparkContext

conf = SparkConf() \
        .setAppName("MySparkApp") \
        .setMaster("local")

sc = SparkContext(conf=conf)
```

---

# Spark Shell and SparkContext

In Spark Shell:

```python
sc
```

SparkContext is already created automatically.

No need to create it manually.

---

# Stopping SparkContext

When the application finishes:

```python
sc.stop()
```

---

## Why Stop It?

Releases:

* Memory
* CPU Resources
* Network Connections

Similar to closing a file after using it.

---

# Limitations of SparkContext

In Spark 1.x:

To use different Spark modules, separate contexts were required.

### Example

```text
SparkContext
SQLContext
HiveContext
StreamingContext
```

Managing multiple contexts became complex.

---

# 2. SparkSession

## Definition

**SparkSession** is the unified entry point introduced in Spark 2.0.

It combines all Spark functionalities into a single object.

---

# Why SparkSession Was Introduced

Before Spark 2.0:

```text
SparkConf
     ↓
SparkContext
     ↓
SQLContext
     ↓
HiveContext
     ↓
StreamingContext
```

Too many contexts had to be created and managed.

---

## Solution

Spark 2.0 introduced:

```text
SparkSession
```

One object for everything.

---

# SparkSession Architecture

```text
           SparkSession
                  │
    ┌─────────────┼─────────────┐
    │             │             │
SparkContext   SQL APIs     Hive APIs
                              │
                        Streaming APIs
```

---

# Features of SparkSession

## 1. Unified Entry Point

Single interface for:

* Core Spark
* SQL
* Hive
* Streaming
* DataFrames

---

## 2. Simplified Development

No need to manage multiple contexts.

---

## 3. Built-in SparkContext

SparkSession internally contains SparkContext.

```text
SparkSession
      ↓
SparkContext
```

---

## 4. Runtime Configuration

Allows modifying Spark properties dynamically.

---

# Creating SparkSession

## Example

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
        .appName("MyApplication") \
        .master("local") \
        .getOrCreate()
```

---

# Using SparkSession

### Read Data

```python
df = spark.read.csv("employees.csv")
```

---

### Display Data

```python
df.show()
```

---

### Run SQL

```python
df.createOrReplaceTempView("emp")

spark.sql(
    "SELECT * FROM emp"
).show()
```

---

# SparkContext vs SparkSession

| Feature           | SparkContext          | SparkSession        |
| ----------------- | --------------------- | ------------------- |
| Introduced        | Spark 1.x             | Spark 2.x           |
| Purpose           | Core Spark Operations | Unified Entry Point |
| SQL Support       | Requires SQLContext   | Built-in            |
| Hive Support      | Requires HiveContext  | Built-in            |
| Streaming Support | Separate Context      | Built-in            |
| Ease of Use       | Complex               | Simple              |
| Recommended Today | No                    | Yes                 |

# Interview Questions

### Q1. What is SparkContext?

SparkContext is the primary entry point that connects a Spark application to a cluster.

---

### Q2. What is SparkConf?

SparkConf is used to store Spark application configurations.

---

### Q3. Why is SparkContext required?

It enables communication between the Spark application and the cluster.

---

### Q4. What is SparkSession?

SparkSession is a unified entry point introduced in Spark 2.0 that combines SparkContext, SQL, Hive, and Streaming functionality.

---

### Q5. Does SparkSession contain SparkContext?

Yes.

```python
spark.sparkContext
```

returns the underlying SparkContext.

---

### Q6. Which is preferred today?

SparkSession.

---

### Q7. What command stops SparkContext?

```python
sc.stop()
```

---

### Q8. Why was SparkSession introduced?

To eliminate the need for multiple contexts such as SQLContext, HiveContext, and StreamingContext.
