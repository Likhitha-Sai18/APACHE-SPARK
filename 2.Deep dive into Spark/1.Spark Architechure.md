# Apache Spark Architecture – Detailed Structured Notes

## Introduction

Apache Spark follows a **Master–Worker Architecture** and consists of three major components:

1. **Driver Program**
2. **Cluster Manager**
3. **Worker Nodes (Executors)**

These components work together to execute Spark applications in a distributed environment.

---

# Apache Spark Architecture

```text
                    Driver Program
                           │
                           │
                    Cluster Manager
                           │
          ┌────────────────┴───────────────┐
          │                                │
      Worker Node 1                  Worker Node 2
          │                                │
      Executor 1                      Executor 2
```

---

# Components of Spark Architecture

## 1. Driver Program

### Definition

The **Driver Program** is the brain of a Spark application.

It is responsible for:

* Running the main Spark application
* Creating SparkContext
* Building execution plans
* Dividing jobs into tasks
* Sending tasks to executors
* Collecting results

---

## Responsibilities of Driver Program

### 1. Execute User Code

The code written by the user runs inside the Driver Program.

Example:

```python
df = spark.read.csv("employees.csv")
df.filter(df.salary > 50000).show()
```

The driver interprets these instructions.

---

### 2. Create SparkContext

The first thing a Spark application does is create a **SparkContext**.

### What is SparkContext?

> SparkContext is the entry point to a Spark Cluster.

It establishes communication between:

```text
Application ↔ Spark Cluster
```

---

### SparkContext in Spark Shell

When using Spark Shell:

```python
sc
```

SparkContext is automatically created.

---

### SparkContext in Spark Code

When writing Spark applications:

```python
from pyspark import SparkContext

sc = SparkContext()
```

SparkContext must be explicitly created.

---

## Spark Shell vs Spark Code

| Spark Shell                    | Spark Code                    |
| ------------------------------ | ----------------------------- |
| Interactive                    | Standalone Application        |
| Used for testing               | Used for production           |
| Executes commands immediately  | Executes complete program     |
| SparkContext already available | SparkContext created manually |

---

### Spark Shell Example

```python
sc.parallelize([1,2,3,4]).collect()
```

Immediate output.

---

### Spark Code Example

```python
from pyspark import SparkContext

sc = SparkContext()

data = sc.parallelize([1,2,3,4])

print(data.collect())
```

Runs as an application.

# 2. DAG (Directed Acyclic Graph)

## Definition

DAG stands for:

**Directed Acyclic Graph**

It is the execution plan generated by Spark.

---

### Directed

Tasks have a direction of execution.

```text
Read Data
    ↓
Filter
    ↓
Sort
    ↓
Save
```

---

### Acyclic

No circular loops.

❌ Invalid:

```text
A → B → C → A
```

✅ Valid:

```text
A → B → C
```

---

## Why DAG?

Spark uses DAG to:

* Optimize execution
* Reduce unnecessary computations
* Improve performance

---

## Example

User Code:

```python
df.filter()
  .groupBy()
  .sort()
```

DAG Generated:

```text
Read Data
      ↓
Filter
      ↓
GroupBy
      ↓
Sort
      ↓
Output
```

---

# 3. Stages

After creating the DAG:

Spark divides it into multiple **Stages**.

---

## Definition

A Stage is a collection of similar tasks executed together.

---

Example:

```text
Stage 1
---------
Read Data
Filter Data

Stage 2
---------
GroupBy
Sort
```

---

### Why Stages?

Stages improve:

* Parallel Processing
* Resource Utilization
* Performance

---

# 4. Tasks

## Definition

A Task is the smallest unit of execution in Spark.

---

### Execution Flow

```text
Application
     ↓
Job
     ↓
Stage
     ↓
Task
```

---

### Example

Suppose data has 4 partitions:

```text
Partition 1 → Task 1
Partition 2 → Task 2
Partition 3 → Task 3
Partition 4 → Task 4
```

All tasks run in parallel.

---

# 5. Cluster Manager

## Definition

The Cluster Manager is responsible for allocating resources to Spark applications.

---

### Responsibilities

* Resource Allocation
* Executor Creation
* Resource Monitoring
* Cluster Coordination

---

## Supported Cluster Managers

### Spark Standalone

Built-in cluster manager.

### Hadoop YARN

Most common in Hadoop environments.

### Kubernetes

Container-based deployment.

### Apache Mesos

Distributed cluster manager.

---

## Working

Driver requests resources.

```text
Driver
   ↓
Cluster Manager
   ↓
Allocates Executors
```

---

# 6. Worker Nodes

## Definition

Worker Nodes are machines that execute Spark tasks.

---

### Responsibilities

* Run executors
* Process data
* Store intermediate results

---

Example:

```text
Cluster
   │
 ┌─┴──────────┐
 │            │
Worker 1   Worker 2
```

---

# 7. Executors

## Definition

Executors are processes running inside worker nodes.

---

### Responsibilities

### Execute Tasks

```text
Task → Executor → Result
```

---

### Store Data

Keep intermediate data in memory.

---

### Return Results

Send output back to the Driver.

---

## Example

```text
Worker Node
     │
 ┌─────────┐
 │Executor │
 └─────────┘
```

---

# Complete Spark Application Lifecycle

## Step 1: User Submits Code

```python
df.filter().groupBy()
```

↓

## Step 2: Driver Program Starts

Creates SparkContext.

↓

## Step 3: DAG Creation

Execution plan generated.

↓

## Step 4: DAG Divided into Stages

```text
Stage 1
Stage 2
Stage 3
```

↓

## Step 5: Stages Converted into Tasks

```text
Task 1
Task 2
Task 3
Task 4
```

↓

## Step 6: Driver Requests Resources

Request sent to Cluster Manager.

↓

## Step 7: Cluster Manager Launches Executors

Executors created on Worker Nodes.

↓

## Step 8: Tasks Sent to Executors

```text
Driver
   ↓
Executors
```

↓

## Step 9: Executors Execute Tasks

Data processed in parallel.

↓

## Step 10: Results Returned

```text
Executors
    ↓
Driver
```

↓

## Step 11: Final Output Generated

Application completed.

# Interview Questions

### Q1. What are the main components of Spark Architecture?

* Driver Program
* Cluster Manager
* Worker Nodes
* Executors

---

### Q2. What is SparkContext?

SparkContext is the entry point of a Spark application to the cluster.

---

### Q3. What is DAG?

Directed Acyclic Graph, an execution plan created by Spark.

---

### Q4. What is a Stage?

A collection of tasks executed together.

---

### Q5. What is a Task?

The smallest unit of execution in Spark.

---

### Q6. What is the role of the Cluster Manager?

Resource allocation and executor management.

---

### Q7. What is an Executor?

A process running on a worker node that executes tasks.

---

### Q8. What is the difference between Driver and Executor?

| Driver                 | Executor        |
| ---------------------- | --------------- |
| Creates execution plan | Executes tasks  |
| Schedules jobs         | Processes data  |
| Collects results       | Returns results |
